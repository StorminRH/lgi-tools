## Implementation blueprint

### Owned surfaces

- `package.json` — sole owner of the exact inner `verify` sequence.
- `docs/workflows/close-out.md` — sole owner of the exact pinned invocation and
  canonical execution/evidence order.
- Shared/public/transitional guidance, both close-out adapters, and
  `.agent-local/policy-manifest.json` — semantic alignment without another exact
  invocation or adapter thinning.
- Master plan, both contracts, this plan, and `docs/SCRATCHPAD.md` — lifecycle
  truth and the two-commit handoff.
- `.agent-local/codegraph_guard.py` — inherited operator-authored improvement,
  verified and delivered without wording edits.

### Interfaces and contracts

- `pnpm verify` — package-owned sequence of typecheck, lint, one coverage-enabled
  Vitest suite, and Fallow, with no standalone non-coverage suite segment.
- Canonical close-out invocation — exactly one pinned invocation in
  `docs/workflows/close-out.md`; its environment reaches nested Fallow.
- `pnpm test` — remains the non-coverage development command and retains focused
  Vitest-argument use.
- Pre-PR design review — consumes current implementation/local evidence and
  returns fixes before final-head verification; its review questions do not change.
- Codegraph hook — continues reading JSON stdin and returning
  `PreToolUse.additionalContext`; unfamiliar areas say `explore`, known symbols say `query`.
- No production export, route, schema, or runtime interface changes.

### Control and data flow

Development runs focused checks as useful. Close-out performs judgment and
local proof while services are available, then final-session design review.
After all fixes and delivery records settle, it shuts services down and runs the
single pinned `verify`, followed by applicable workflow checkers, commit, and
push. Evidence is attached to the tested HEAD. PR opening consumes it unchanged;
later edits dispatch only the gates their changed inputs invalidate. Runtime
application data flow does not change.

### Edge and failure behavior

- Verification failure → keep `coverage/` for diagnosis; remove it only after a
  successful final pass.
- Documentation-only/lifecycle-only change → rerun its document, release,
  drift, privacy, and diff checks, not application tests.
- Package, script, code, test, or verification-configuration change → rerun the
  full pinned gate.
- Incomplete operator hook → syntax or smoke evidence fails and blocks delivery.
- Specialized `VERSION_AUDIT` and `CONVEX` commands → remain independent and unchanged.
- Session B planning → treats session A's command/policy alignment as a delivered dependency.
