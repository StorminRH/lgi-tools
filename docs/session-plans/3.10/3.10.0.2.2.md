## Resolved implementation decisions

- **Contract PD-1 — How the checker reads the baseline schema: runtime parse of the
  canonical template into a structured schema.** `check_baseline_claims` parses
  `docs/workflows/schema/code-health-baseline.md` at runtime into a structured
  `BaselineSchema` — the allowed `##` section set, the allowed identity keys, the
  canonical metric-key registry, and the metric-table column names — via new
  `parse_baseline_schema(root) -> BaselineSchema | None` (fail-closed `None` on a
  missing/unparseable template). All enforcement derives from that parsed schema, so
  the form is the single owner (no allowed keys or metric registry hard-coded in the
  checker). Exact-schema integrity is locked by a build-time test comparing the parsed
  schema to the canonical expectation. **Rejected:** deriving only section titles and
  hard-coding identity/metric grammar in the checker (duplicates the schema — the
  defect the re-review flagged); **rejected:** a generated `policy-manifest.json`
  entry (duplication + a generator, against "derived, never duplicated").

- **Contract PD-2 — Where/when version-start is captured and how immutability is
  enforced: same-version freeze against `origin/main`, adoption deferred to 0.5.**
  The frozen anchor is the version-start column as committed on `origin/main`. A new
  seam `frozen_version_start(root, *, read=_git_show_main) -> BaselineAnchor` parses
  `git show origin/main:docs/CODE_HEALTH_BASELINE.md` into an ordered
  `{metric-key: version-start cell}` map, tagged `enforced`, `bootstrap`, or
  `unavailable`. `_version_start_findings`:
  - **`enforced`** (main carries a version-start column) → the working version-start
    key-set **and** every value must be identical to main; any added/removed/renamed
    key or changed value → `error`. Within a version the frozen column is invariant on
    `origin/main`, so a stale-but-readable ref yields the same values — staleness is
    harmless and no freshness proof is needed.
  - **`bootstrap`** (main has no version-start column — the pre-migration old format)
    → a distinct one-time condition permitting this PR's reformat; it can never
    re-trigger once main carries the column.
  - **`unavailable`** (no work tree, `git show` error, unreadable ref) → a **blocking
    `error`** (fail-closed); enforcement runs at close-out/drift in-repo where
    `origin/main` exists, and unit tests inject `read`, so there is no fail-open
    surface.

  The **version-adoption write-exception** — allowing 0.5's audit to write the *next*
  version's frozen column at version close — is **not** built here. `active_roadmap`
  still returns the *current* version at close, so no correct adoption signal exists
  in this session; 0.5, which performs the sanctioned write, owns adding the exception
  (part of "writing the baseline at adoption," OOS-2). For all of v3.10, same-version
  enforcement + bootstrap is complete and correct. **Rejected:** the earlier
  `APP_VERSION`- or `active_roadmap`-keyed adoption signal (release-line vs lifecycle
  mismatch; and `active_roadmap` cannot name the next version before archival — both
  unsound per re-review); **rejected:** fail-open-on-git-failure; **rejected:** a
  separate frozen snapshot file (duplication + needs 0.5's write).

### Interfaces and contracts

New/changed exports in `check_baseline_claims.py` (each new function carries a
docstring stating contract, units, preconditions, and caller ownership):

- `BASELINE_TEMPLATE_RELPATH = "docs/workflows/schema/code-health-baseline.md"` —
  module constant; single source of the allowed schema (drift checks its existence).
- `parse_baseline_schema(root: Path) -> BaselineSchema | None` — *"Parse the baseline
  template into its structured schema: allowed section titles, allowed identity keys,
  the canonical metric-key registry, and the metric column names. Return `None` when
  the template is unusable (absent, or missing a required structural part) so callers
  treat it as a blocking configuration error and schema enforcement never silently
  disables. Full-schema integrity against the canonical expectation is locked by a
  build-time test."*
- `frozen_version_start(root: Path, *, read: Callable[[Path], str | None] = _git_show_main) -> BaselineAnchor`
  — *"Return the version-start anchor from `origin/main`: an ordered {metric-key:
  version-start cell} map tagged `enforced` (main carries the column), `bootstrap`
  (main lacks it — the pre-migration old format), or `unavailable` (git/read failure).
  `unavailable` is a blocking error for the caller; `bootstrap` permits the one-time
  reformat; `enforced` requires byte-identical key-set and values. `read` is
  injectable so tests need no git."*
- `_schema_only_findings(root, lines, schema) -> list[Finding]` — error on any `##`
  section ∉ `schema.sections`; any non-blank line that is not a heading, table
  row/separator, watch-trigger fence line, or `Watch (AF-nnn)` bullet; any metric row
  without exactly four cells, with a key ∉ `schema.metric_keys`, with a duplicate key,
  or with a delta ≠ the derived value (signed integer difference for bare-integer
  metrics, `—` otherwise); any identity row whose key ∉ `schema.identity_keys`;
  any required registry metric absent.
- `_version_start_findings(root, rows, anchor) -> list[Finding]` — error per PD-2's
  `enforced` case; no findings under `bootstrap`; the `unavailable` blocking error is
  raised where the anchor is obtained.
- Repointed: `_current_cell` and the row parsers read the **Current** cell at its new
  index; export assertions in `_named_file_claims` come only from the Current cell;
  `_auth_surface_claim` reads the `auth-surface files` metric row (Rails prose is gone).

The changelog and baseline **forms** are authored documents, not code exports. No
`src/`/`convex/` production export changes (HC-3); `app-version.ts` is a constant bump.

### Control and data flow

`check_baseline_claims.collect_findings(root)` (unchanged entry point):
1. Load `docs/CODE_HEALTH_BASELINE.md`; missing → single error (unchanged).
2. `schema = parse_baseline_schema(root)`; `None` → single blocking error naming the
   template (fail-closed).
3. `anchor = frozen_version_start(root)`; `unavailable` → single blocking error.
4. Collect: `_step_one_claims`, `_named_file_claims`, `_auth_surface_claim` (repointed
   to Current), `_schema_only_findings(..., schema)`, `_version_start_findings(...,
   anchor)`.
5. `run_checker` prints JSON `{errors, warnings}`; under `--check`, a nonzero exit
   follows any error. `check_agent_drift`'s `check_lifecycle_checkers` runs this
   checker among `LIFECYCLE_CHECKERS`, promoting its `errors` to blocking drift
   failures and keeping stale-count `warnings` advisory (the promotion mechanism is
   already covered by `test_agent_drift.py`).

`frozen_version_start` shells `git show origin/main:...` via `subprocess` (in-idiom
with `repo_measures.py`/the resolver); a failure yields `unavailable`, not a silent
skip. No network, no writes.

### Edge and failure behavior

- **This PR's own reformat (main old-format)** → `bootstrap` → permitted; after merge
  main carries the frozen column and later v3.10 sessions are `enforced`.
- **Mid-version version-start edit / key rename / delete / add** → `enforced`, key-set
  or value differs → `error`.
- **Version adoption (v3.11 open, 0.5's audit)** → out of scope here; 0.5 adds the
  exception when it builds the baseline-writing audit.
- **git/read failure at enforcement** → `unavailable` → blocking error (fail-closed).
- **Missing/unparseable template** → `parse_baseline_schema` `None` → blocking error.
- **Prose row masquerading as a metric** (`| Notes | … | … | — |`) → key `Notes` ∉
  `schema.metric_keys` → `error`.
- **Watch symmetry** → the reformat keeps `AF-006/007/008` triggers **and** their
  `Watch (AF-nnn)` carriers, so `check_lifecycle_evidence._baseline_symmetry_findings`
  stays green; dropped `Verified` carriers + campaign queue are inert
  (`_audit_baseline_findings` only fires when `docs/version-audits/<active>/PLAN.md`
  exists — none does).
