## Implementation blueprint

### Owned surfaces

- `docs/workflows/` — sole owner of shared executable workflow ordering and judgment.
- `.agents/skills/` and `.claude/skills/` — thin runtime invocation and mechanics adapters.
- `AGENTS.md`, `src/AGENTS.md`, and `CONTRIBUTING.md` — repository, source, and human contributor maps with non-overlapping audiences.
- `.agent-local/policy-manifest.json` and `check_agent_drift.py` — declared procedure ownership, dependency reconciliation, and deterministic prose-ownership enforcement.
- `docs/workflows/schema/session-contract.md` and resolver validation — execution-bundle artifact form and fail-closed field checks.

### Interfaces and contracts

- Contract execution frame — exact `Execution profile` and `Delivery unit`; non-empty roadmap coverage, ordered phases, and split triggers.
- `pairedSkills.<name>.procedure` — one repository-relative canonical workflow path required once in each runtime adapter.
- `procedurePolicies.<path>.orderedRequired` — ordered regex fragments that preserve plan-version topology review.
- `proseOwnership` — explicit scan paths, minimum word count, and exact sentence/path/reason exceptions.
- No production export, API, route, schema, or runtime data contract changes.

### Control and data flow

Lifecycle dispatch remains resolver → one skill adapter → one canonical
procedure → fixed result/handoff. Planning flows roadmap outcomes → topology
audit and approval → roadmap topology update → one contract per bundle → one
session plan. Drift verification reads the manifest, validates adapters and
procedures, extracts normalized prose from declared owners, and fails on any
unexcepted cross-file duplicate.

### Edge and failure behavior

- Missing prerequisite or materially different resolver directive → stop without inventing a branch or stacked substitute.
- Missing/malformed execution-frame marker → `contract-repair-needed` with the selected contract named.
- Plan-version checkpoint missing or reordered → drift fixture and live gate fail before contracts can be trusted.
- Duplicate normalized sentence → sorted path/line diagnostic; no fuzzy or broad exception fallback.
- Wording change after checkpoint → rerun affected checks, push a new checkpoint, and pause again.
- Merge authority absent after review-ready PR → return BLOCKED with the PR open.

### Ordered work

1. **Reconcile lifecycle topology.** Preserve the roadmap rider, collapse Phase Zero to 3.10.0.4.1, replace its contract/index, remove superseded contracts, and add this approved plan.
2. **Redesign planning first.** Add contract execution framing and the canonical plan-version/plan-session procedures with enforced topology-review order and no silent re-splitting.
3. **Canonicalize remaining workflows.** Extract start-session, design review, audit, watch, resolve-watch, UX, and triage behavior; preserve close-out; thin both adapter trees.
4. **Consolidate maps and retire guides.** Move source rules to `src/AGENTS.md`, keep root concise, align only the named contributor drift, archive and remove retired guides, and redirect live references.
5. **Rebuild policy enforcement.** Add procedure mappings, ordered procedure policies, exact-normalized prose ownership, resolver validation, fixtures, and regenerated reconciliation hashes.
6. **Reach the wording gate.** Run design review and focused checks, commit/push the checkpoint, return the complete review packet with no PR, and wait.
7. **Finalize after approval.** Prepare truthful release state, run the sole final verification, open and triage one PR to review-ready state, and stop unmerged.
