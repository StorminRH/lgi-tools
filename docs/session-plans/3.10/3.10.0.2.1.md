## Context

Phase 0 of v3.10 "Hull Integrity" stops treating docs as the medium of
enforcement: every rule routes to one owner — the **resolver/checkers** (CHECK),
the **skills** (STEP), or a small **map** set (MAP). This session (A of two on
one branch for sub-version 3.10.0.2) builds the **canonical frame** the later
migrations pour content into, and retires the two docs the resolver has already
made redundant.

`docs/DEVELOPMENT_LIFECYCLE.md` and `docs/SESSION_CONTRACTS.md` are prose
describing what the resolver already enforces. A `gpt-5.6-sol` section-home audit
confirmed that **every section's machine-verifiable behavior already lives in
`.agent-local/`**; deletion removes redundant prose, not enforcement. The two
sections the contract flags (`DEVELOPMENT_LIFECYCLE` §2 runtime-todo, §8
health-feedback) are, by nature, STEP/MAP content, and their essence survives in
the homes the master plan's model assigns them (see "Deletion-safety").

Today the resolver validates a selected contract **only** by its `UX gate:`
marker; it never checks that the 12 required sections exist. This session adds
that parser, deriving the required sections from a new canonical template's own
headers, and routing a structurally-broken contract to an **operator-stop repair
directive** — while making a *missing/unparseable template itself* a blocking
`invalid` state so validation can never silently fail open.

This is a **planning session**: the only artifact written on approval is the
session plan. All implementation below happens in a later execution session.

## 5. Interface designs (Python; equivalent of the /** */ interface standard)

Module constants in `resolve_development_state.py`:

```python
# Canonical contract template whose `## <n>. <title>` headers are the single
# source of the required contract sections (derived, never duplicated).
CONTRACT_TEMPLATE_RELPATH = "docs/workflows/schema/session-contract.md"

# The pre-dispatch gate a runtime must pass before dispatching any handler.
# Must equal developmentState.preDispatchGate in the manifest (drift-checked).
RELEASE_CONSISTENCY_GATE = "python3 .agent-local/check_release_consistency.py --check"
```

New functions (fail-closed, missing-file-safe):

```python
def required_contract_sections(root: Path) -> list[str] | None:
    """Return the required contract section titles, or None when unusable.

    Parses `## <int>. <title>` headers from CONTRACT_TEMPLATE_RELPATH and
    returns each <title> (leading `<int>. ` stripped), in template order.
    Returns None when the template is UNUSABLE — absent, zero parseable numbered
    headers, DUPLICATE titles, or NON-CONTIGUOUS numbering (headers not 1..K) —
    so partial corruption (an 11-header truncation, a dropped/duplicated header)
    cannot silently weaken enforcement. The caller treats None as a BLOCKING
    invalid state. Exact-count integrity (== the 12 canonical titles) is locked
    separately by a build-time test (Step 9), not hard-coded in the resolver.
    """

def contract_section_titles(contract: Path) -> list[str]:
    """Return the contract's `## ` header titles (leading `<int>. ` stripped).

    Returns [] when the contract file is absent, so a missing indexed contract
    keeps its existing 'points to missing …' error path and is NEVER recast as a
    structural-repair (a missing FILE is not a missing SECTION).
    """
```

`WorkflowDirective.as_dict` gains one **derived** key (no new dataclass field):

```python
# present for any dispatchable directive, null for operator stops (handler None),
# mirroring `handler`. Value is drift-checked against manifest.preDispatchGate.
"preDispatchGate": RELEASE_CONSISTENCY_GATE if self.handler is not None else None,
```

New resolved states + their `directive_for` branches:

- `contract-repair-needed` — the selected contract is missing required
  section(s). State carries `missingContractSections: list[str]`, `session`,
  `subversion`, `contract`. Directive: operator stop (handler=None, mode=report,

## 6. Design alternatives (selected vs rejected)

- **Header derivation — runtime parse (SELECTED)** vs generated-manifest entry.
  Master plan mandates derivation "from the template's own headers … derived,
  never duplicated." A manifest copy duplicates + needs a generator — rejected.
- **Missing/unparseable template — blocking `invalid` (SELECTED)** vs
  fail-open-skip. Skipping lets the resolver dispatch with validation silently
  off if the template is ever deleted/emptied (start-session runs the resolver
  before the drift gate). A blocking invalid makes the template a hard resolver
  dependency; fixtures install a real template. Rejected: silent skip.
- **Repair routing — operator-stop / null handler (SELECTED per Ryan)** vs a
  `plan-version` repair handler. Matches the `master-plan-needed` shape; the fix
  rides the active branch. No new handler registration.
- **Missing-section as a clean stage, not an error (SELECTED)** — follows the
  "missing UX gate marker" precedent (directive, no resolver error). Rejected:
  hard-erroring on a missing section.
- **A9 — mechanically-locked `preDispatchGate` directive key (SELECTED)** vs the
  resolver running `check_release_consistency` inline (couples checkers, breaks
  the resolver's pure-reader purity) OR a prose-only field (unenforced). The
  selected design: derive the key from handler, declare the expected command in
  the manifest, and have `check_development_state` validate the value +
  handler/null relationship, plus a `preDispatchGate` regex in start-session's
  paired-required so the skill must consume it.
