## Why now

Session A consolidated the close-out *content* into one canonical procedure but
deliberately left the sources and the two thick adapters in place so nothing
lost a home mid-flight. Until session B removes the sources, thins the adapters,
and repoints the enforcement machinery, the corpus still states the close-out
rules in five places and the drift gate still polices the copies. Shipping this
retirement is what makes `docs/workflows/close-out.md` the *single* owner and
closes sub-version 3.10.0.3 — the Phase-0 exit condition for the close-out
workflow.

## Scope (the destination)

The finished slice leaves exactly one close-out owner. `docs/workflows/close-out.md`
is the sole shared sequence and additionally gains one bounded refinement: an
explicit finalized-diff boundary preflight and a reordering that runs the cheap,
edit-capable workflow checks before the single pinned `pnpm verify`, leaving only

## Resolved implementation decisions

- **Contract PD-1 — shared/runtime boundary: the canonical procedure owns every
  shared step and all ordering; each adapter keeps only invocation authority plus
  genuine per-runtime mechanics.** The adapter retains: (a) the one-line pointer
  to `docs/workflows/close-out.md` as the sole sequence with a statement that it
  owns ordering; (b) the conditional per-run merge authorization (current-head
  Greptile 5/5, zero unresolved findings, green CI, mergeable/CLEAN PR — no gate
  bypass, no unrelated production action); (c) native task-list creation
  (Claude Code task list / Codex task list); (d) background-launch of the
  procedure's PR-gate poll (`run_in_background` Bash / Codex background job); and
  (e) the runtime inline-reply watch the procedure's justification step defers to
  (`close-out.md:168`). Everything else — baseline/watch checkers, PR headings,
  scrub, coverage checkpoint, Greptile loop, merge, after-merge reconciliation,
  resolver handoff — is deleted from both bodies and read from the procedure.
  **Rejected:** keeping a "summary of shared steps" in each adapter for
  convenience; that reintroduces the duplication the sub-version exists to remove
  and would fail the 3.10.0.7 anti-duplication check being built on top of this.
- **Contract PD-2 — surviving-doc references: redirect, do not allowlist.** The
  only live (non-RECORD) surfaces with backticked path claims to the deleted docs
  are `AGENTS.md` (lines 225, 231, 237, 290–292) and
  `docs/PRE_PR_DESIGN_REVIEW.md` (lines 6, 146); both redirect to
  `docs/workflows/close-out.md`. `check_doc_refs.py` already exempts the RECORD
  sources that also mention the docs — `docs/VERSION_*_PLAN.md`,
  `docs/SCRATCHPAD.md`, `session-plans/**`, `session-contracts/**` — and those
  are frozen or living-state history that must not be rewritten. No `_ALLOWLIST`
  entry references a deleted doc, and none needs adding because every live
  reference points at a file that exists after the change. The contract's
  "`check_doc_refs.py` allowlist" item therefore resolves to an explicit audit
  with no code change, proven by a clean run (SC-2). **Rejected:** adding a
  historical `_ALLOWLIST` exemption for the deleted docs — that would preserve a
  dead pointer instead of sending readers to the real owner.

### Edge and failure behavior

- A shared-step regex left in `pairedSkills.close-out.required` after thinning →
  `check_agent_drift.py` fails (adapter no longer contains it) → remove the regex
  before restamping. Owning layer: manifest.
- `reconcile_skill_ledger.py` run before `AGENTS.md` is final → hashes go stale on
  the next edit → restamp is the last policy step, after all guide/adapter/manifest
  edits settle. Owning layer: Ordered work step 5.
- A backticked `docs/<deleted>.md` claim missed in a live surface → `check_doc_refs.py`
  errors → redirect it (SC-2 gates this). RECORD sources are exempt by design and
  are not edited.
- `APP_VERSION` bump touches `src/` → invalidates the full `pnpm verify` gate →
  the single pinned checkpoint runs after the bump (it is a final-head record).
- Stray untracked ` 2.md` duplicates (`docs/workflows/close-out 2.md`,
  `docs/session-plans/3.10/3.10.0.3.1 2.md`) → local cruft, deleted during the
  clear-local-artifacts step so they never enter the commit or confuse the
