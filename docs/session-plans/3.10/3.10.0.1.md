## 5. Interface designs (Step 3 answers + draft doc comment)

No new TS/`convex/` export (the `APP_VERSION` export already has its interface
comment; only its value changes). New surface = the checker's RECORD exemption:

**`_RECORD_SOURCES` (constant) + `_is_record_source(source)` (predicate)**

- **Decision it owns (P2):** *which scanned documents are RECORD* — frozen history
  (plans/contracts/audits, SCRATCHPAD) and machine-owned living state (the active
  roadmap, backlog, code-health baseline). One owner beside `_ALLOWLIST` /
  `_ARCHIVE_REDIRECTS`, so all of this checker's exemptions live in one file
  (Ryan's decision #1: checker-local).
- **Surface (P1):** tuple of `(source_glob, reason)` pairs, `fnmatch`ed against the
  scanned file's repo-relative POSIX path. The changelog dir is **not** included:
  `_source_paths` never scans `content/`, so an entry there would be dead (P4).
- **Real caller (P4):** exactly one — the terminal error branch of `collect_findings`.
- **Behavior (P5):** for a RECORD source, the *"repository path does not resolve"
  error is suppressed*; the **archive-redirect and `../` warning branches still
  run**. RECORD documents are not rewritten to satisfy the checker, and their path
  validity is owned by their real validators (resolver / `check_baseline_claims`),
  which is why generic path-policing is the wrong tool here (P10: the rail is
  adjusted deliberately, with rationale).
- **Invariant (P7):** the doc comment states this suppresses only plain repo-path
  errors, never warnings, and names *why* RECORD is machine-owned rather than
  path-policed.

Draft docstring:

    def _is_record_source(source: str) -> bool:
        """Return whether a scanned document is RECORD (exempt from path errors).

        RECORD is frozen history (prior/current plans, contracts, audits, and
        SCRATCHPAD) and machine-owned living state (the active roadmap, backlog,
        and code-health baseline). These legitimately reference archived, deleted,
        or not-yet-created paths, are never rewritten to satisfy this checker, and
        have their real validity owned by the resolver and check_baseline_claims —
        not by generic path existence. Only the plain unresolved-repository-path
        error is suppressed; archive-redirect and ``../`` warnings still fire.
        """

`_RECORD_SOURCES` globs — frozen: the session-plans, session-contracts, and
version-audits trees plus `docs/SCRATCHPAD.md`; living: the active
`VERSION_*_PLAN` roadmap, `docs/backlog.md`, and `docs/CODE_HEALTH_BASELINE.md`.

Insertion (verified against the checker's terminal error branch): gate only the
final error append with `and not _is_record_source(source)`, leaving the
archive-redirect and relative-reference warning branches untouched. This clears
all 56 doc-ref errors (every erroring file is a RECORD source) and the latent
SCRATCHPAD one, while leaving `docs/CONVEX.md`, `AGENTS.md`, the guides, and the
skills fully policed.

## 6. Design alternatives (Design-it-twice)

**A — exempt scope (the two-pass pivot; settled to match the contract).**
- *Chosen:* exempt **frozen + living** RECORD from *repo-path errors* (contract
  §3), refined to **suppress the error only, preserve warnings**.
- *Rejected:* frozen-only suppression + a per-token roadmap allowlist (an
  adversarial-review suggestion). Rejected on reflection because it deviates from
  the contract (§3 exempts living state; §7 asks only *which* living roots, not
  *whether*), would require a `plan-version` / contract reapproval for a marginal
  gain, and the "typo-detection" it preserves is doc-ref checking of living state
  — which the master plan explicitly reassigns to purpose-built checkers. Keeping
  the archive warnings (not a whole-file skip) captures the one real safety point
  the review raised without the deviation.

**B — exemption home (settled with Ryan).** Checker-local constant (chosen) vs a
manifest key (rejected: splits exemptions, single consumer, P4).
