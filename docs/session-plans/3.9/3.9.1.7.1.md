## Interface designs

**No production exports are created or changed.** The only new artifacts are
workspace-local scratch tooling in an **untracked**
`.agent-local/comment-sweep/` directory (deleted at sub-version close per
scratch-script hygiene):

- `convert_comments.py` (Python 3 stdlib, arc idiom) — modes:
  - `--list`: print conversion sites (file:line) without writing;

  - `--check`: prose-equivalence proof — strip comment syntax before/after

- A self-contained dry-run mini-package (own `package.json` +
  `node_modules`, untracked) holding `eslint-plugin-jsdoc`,
  `eslint-plugin-tsdoc`, and `typescript-eslint`, plus `dryrun.config.mjs` —
  the three contracted rules run via `eslint -c` against production `src/` +
  `convex/`. **Session .1's tracked diff is therefore pure comments** — no
  `package.json`/lockfile/`eslint.config.mjs` change until .2 (plugin
  versions recorded in the worklist artifact so .2 installs identical
  versions as devDependencies).

Draft header comment for the script (sample of the standard applied to
tooling):

```python
"""One-shot 3.9.1.7.1 comment-syntax converter: rewrites contiguous `//` blocks
directly above exported declarations in src/ and convex/ production code to
`/** */`, preserving prose byte-for-byte (1:1 line mapping, no re-wrapping).
Never touches blank-line-separated comments, directives, or comment bodies."""
```

### Conversion rules (the decision the script owns)

- **Attachment = contiguity:** only a `//` block with no blank line between
  it and the export converts. The 73 blank-line-separated comments stay `//`
  (module/rationale commentary); their exports land on the worklist for .2's
  judgment.
- **1:1 line mapping, no re-wrap:** single-line `// X` → `/** X */`;
  multi-line block → `/**` + one ` * ` line per source line + ` */`. Prose
  bytes unchanged.
- **Directives never convert:** `eslint-disable*`, `@ts-*`,
  `prettier-ignore` lines are excluded; a prose block above a directive line
  converts and stays above it (1 such site — hand-verified).
- **Flagged, not auto-converted:** any prose line containing `*/` or starting
  with `@`/`{` (TSDoc-significant) is flagged for manual handling during hunk
  review.

## Design alternatives

1. **Conversion tool — chosen: stdlib Python line-based script** (matches the
   arc's tooling idiom, zero dependencies, and the measured surface is clean:
   0 banners, 1 directive adjacency; every hunk is agent-reviewed and the
   `--check` mode proves verbatimness mechanically). *Rejected:*
   TypeScript-AST script (ts-morph/compiler API) — more precise export
   detection in theory, but adds a dependency/runner, has its own
   comment-attachment edge cases, and its precision advantage is moot given
   the measured adjacency facts plus the ESLint cross-check below.
2. **Worklist source — chosen: ESLint dry-run with the real .2 rules**,
   cross-validated against the independent Python scan (discrepancies
   reconciled by hand). The worklist is complete *under the exact rule that
   will gate in .2*, so .2 cannot discover a scope mismatch mid-authoring.
   *Rejected:* scanner-only worklist — simpler, but its export-detection is
   an approximation of the rule that actually flips on, and a mismatch
   surfaces at the worst time.

## Contract §7 decisions (approved resolutions)

1. **Export-detection scope:** the presence rule is `jsdoc/require-jsdoc`
   with `publicOnly: { esm: true }`, `require` for function/class
   declarations, and `contexts` for exported interfaces, type aliases,
   enums, and variable declarations — i.e., **all declaration exports
   including default, type-only, and framework-convention exports**
   (page/layout/route handlers get one-line contract comments in .2; one
   style, no carve-outs). The 34 **re-export statements are not comment
   sites** — the underlying declaration carries the doc and `publicOnly`
   resolves through `export { X }`.
2. **Exemption set:** `**/*.test.{ts,tsx}`, `scripts/**`,
   `convex/_generated/**` (home of the five generated-header suppressions —
   TSDoc validation never sees them), `*.d.ts`. Root configs are out
   automatically (the standard scopes to `src/` + `convex/`). Session
   verifies the five suppressions do live under `_generated/` and adjusts
   only if not.
3. **Line-wrap normalization: no** — verbatim means verbatim; 1:1 line
   mapping as specified above.
4. *(Adjunct)* **TSDoc-syntax trips on converted verbatim prose:** pure
   syntax-escaping (no word changes) is applied in .1 and each site
   recorded; anything genuinely needing rewording is **recorded for .2**,
   which owns authoring judgment — .1 never rewords.
