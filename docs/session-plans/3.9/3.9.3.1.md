## 5. Interface designs

No new or changed exports; no interface comments needed. The two contract
decisions resolve as:

- **Evidence bar for "provably shipped/superseded":** two legs, both required —
  (a) the shipping event named exactly (sub-version + PR # + squash sha), and
  (b) a live-repo mechanical check run this session (file/rule present or
  absent, e.g. the lint rule id, the wrapper module, `find` absence). An entry
  with only one leg is NOT deleted; it gets a corrected trigger instead.
  *Local-state variant* (gitignored artifacts have no shipping commit): leg (a)
  becomes the event that closed their reason to exist (the fallow trial closed
  at adoption, PR #116 `81f8cd9`), leg (b) stays the live absence check.
- **Supersession-evidence placement (contract acceptance: "the diffed backlog
  shows each deletion with its supersession evidence"):** each deletion is its
  own commit whose message names the supersession event, so the backlog diff +
  `git log` pair is the durable record; the session evidence additionally
  carries a per-entry dispositions table. No tombstone text stays in the file.
- **Dispositions-report format:** a short titled block presented in chat at
  session end and recorded in `docs/SCRATCHPAD.md`'s Now section (operator
  state, not a new tracked doc; PR bodies are public and excluded). Each line:
  entry → why it may belong in 3.9 → suggested path (amendment row or existing
  slice). Ryan acts via the amendment path; not a session blocker.

## 6. Design alternatives

- **Selected:** in-place entry deletion, one commit per deletion with the
  supersession named in the commit message (diff + log = the durable record),
  plus the per-entry dispositions table in session evidence. Backlog stays a
  clean someday-pile.
- **Rejected:** tombstone/ledger restructuring (a "deleted entries" section) —
  bloats the file the contract is trying to shrink; git history already is the
  ledger. One sentence: the diff is the record, so duplicating it inline adds
  reader load for no new information.
