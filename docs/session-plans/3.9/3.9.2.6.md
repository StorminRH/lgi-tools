## 5. Interface designs (draft interface comments)

**`schema-reflection.ts`** (new, in `src/db/test-support/`):

```ts
/**
 * Reflects every Drizzle pgTable across all slice schema files (glob census,
 * deduplicated by SQL table name) so registry gates share one schema-discovery
 * mechanism instead of each re-owning the walk. Test-support only:
 * import.meta.glob is a Vite/Vitest construct and never ships at runtime.
 */
export async function reflectedSchemaTables(): Promise<PgTable[]>
```

- Owns/hides (P2): the schema-walk decision (glob pattern, dedup rule).
- Callers: growth gate (refactored), dataset-declaration gate. No other exports.

**`src/purge/coverage.ts`** — one added export (owner of the identity-column decision):

```ts
/**
 * The key-shape rail: scans every foreign key in the given tables and returns a
 * finding for each FK column that targets an identity table (`user`,
 * `characters`) under a name outside the sanctioned identity set. A novel
 * column name (e.g. `created_by` → user.id) would otherwise slip the
 * name-based user-data scan — and with it every registry gate downstream.
 * DB-free reflection; the dataset-declaration gate feeds it the full schema
 * census and requires zero findings.
 */
export function findIdentityFkLeaks(tables: readonly PgTable[]): string[]
```

- The identity-table name set stays a private const in `coverage.ts` (no schema imports
  needed — FK targets are read via `getTableConfig(fk.reference().foreignTable).name`;
  verified against installed Drizzle: `getTableConfig` exposes `foreignKeys`, and
  `ForeignKey.reference()` yields `columns`/`foreignTable`).
- Failure edge a caller must own: none — pure list-out; empty array = clean (P5).

**`dataset-declarations.test.ts`** (new gate in `src/db/`; test file, no exports). Prologue =
the owning rationale comment recording the Combine-vs-Keep verdict, the P1 counts, and
the per-dataset checklist (key shape → purge claim/retain → growth story → ESI entry when
externally fed). Pure join helpers live in-file (the growth-gate precedent), so red paths
are testable without production surface.

## 6. Design alternatives (recorded verdict)

- **Rejected — Combine (`defineDataset` manifest):** callers keep all three declaration
  vocabularies and additionally learn the manifest shape plus its applicability matrix
  (growth's census covers ~55 durable tables including framework/SDE tables that are not
  datasets) — **5 concepts, nothing removed**; wide grab-bag whose three consumers each
  read one section (P8); three unrelated change axes in one home (P2); the import-free
  ESI entries leaf would absorb Drizzle imports. File win for a new dataset is 4→3.
- **Selected — Keep, indexed registries:** **3 concepts, 0 new**; each registry keeps its
  owner and semantics; a test-only index joins their exported data over one census and
  teaches the checklist at the moment of failure; the FK-closure rail closes the real gap.
- One-sentence seed for the rationale comment: *Combining was rejected because the union
  of declaration vocabularies is invariant under the merge — it adds manifest and
  applicability concepts while removing none, and the three consumers each read one
  section (P8); the index gate delivers the single-checklist benefit without them.*
