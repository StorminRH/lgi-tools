## 4. Interface designs (the three formats + rules)

### 4a. Watch trigger block — owner: baseline Watch rows; spec home: `docs/VERSION_AUDIT.md`

One fenced block per Watch finding in a new `### Watch triggers` subsection of `docs/CODE_HEALTH_BASELINE.md` (fenced blocks can't live in table cells; the hotspot table keeps judgment prose and cites the AF id):

```watch-trigger
AF-006: exports(src/data/telemetry/queries.ts) >= 26
```

**Grammar (closed set, one expression per line; any line true ⇒ tripped):**

- `<expr> := <metric>(<arg>) <op> <integer>`
- `<metric> := exports | files | clones` — `exports(<repo path>)` = `^export` line count; `files(zone:<fallow-zone>)` = files assigned to that zone in `.fallowrc.json`; `clones(<fallow-dup-id>)` = files in that clone group in a whole-version pinned Fallow run
- `<op> := >= | > | <= | < | ==`
- Semantics: **trip-form** — expression true means the trigger fired; the 3.9.1.4 checker prints `promote AF-NNN` as a **warn**; classification stays an audit decision (P10).

Initial blocks: `AF-006: exports(src/data/telemetry/queries.ts) >= 26` · `AF-007: exports(src/data/esi-refresh-jobs/queries.ts) > 15` · `AF-008: files(zone:auth-surface) >= 4` · `AF-009: clones(dup:b54bf337) >= 3`. Judgment fragments ("new axis", "group growth") stay in row prose; the countable threshold lives **only** in the block.

**Contract decision resolved — grammar kinds:** `exports`, `files`, `clones` only. `loc` was considered and dropped: no live trigger uses it (P4); the grammar is a closed set extended only by editing the spec (a shared-policy change).

The audit ledger (`docs/version-audits/X.Y/PLAN.md`) cites the AF id and never restates the trigger; `docs/DEVELOPMENT_LIFECYCLE.md` §1's ownership table records this.

### 4d. Marker vocabulary + legacy exemption

- `Execution status` is exactly `Pending` or `Complete` (stated as exact in SESSION_CONTRACTS.md and DEVELOPMENT_LIFECYCLE §4).
- Roadmap terminal statuses are the exact closed set `SHIPPED | COMPLETE | DEFERRED | CANCELLED`; a status cell contains exactly one vocabulary token — delivery evidence (PR #s) lives in the changelog/SCRATCHPAD, never in the cell. (The resolver's substring match — where `INCOMPLETE` currently reads as terminal — is 3.9.1.2's fix; this session fixes only the doc contract.)
- **Contract decision resolved — legacy exemption: yes.** Formats and vocabularies bind 3.9-and-later artifacts; archived 3.8 material is never retro-edited; checkers treat pre-3.9 artifacts as legacy. Stated once in DEVELOPMENT_LIFECYCLE.

### 4e. Planning-terminal rule

One rule, stated in DEVELOPMENT_LIFECYCLE §5, SESSION_PLANNING Step 9, and the `start-session` + all four `plan-*` skills (both trees): *a session that planned an artifact never executes it.* Plan-mode acceptance authorizes artifact persistence only; after a planning handler persists its approved artifact, `start-session` reports the resolver's new directive and **stops**. Execution begins in a fresh `start-session`, whichever runtime runs it. The four plan-* skills' closing formula changes from "…and return control to `start-session` without predicting the next stage" to "…report its new directive, run the drift check, and stop — the planning session is terminal; execution begins in a fresh `start-session`."

**Meta-note:** the rule ships *in* this session, so this session itself follows current policy — after approval it persists the plan and proceeds to execution in this conversation. The new rule binds from the next planning session onward.

## 5. Design alternatives

| Decision | Selected | Rejected + why |
| --- | --- | --- |
| Trigger expression direction | Trip-form (`>= 26` fires promote) | Invariant-form (`<= 25` must hold — the master-plan sketch): a reader/checker must mentally negate; "trigger" semantically names the promotion event |
| Trigger placement | Fenced blocks in one `### Watch triggers` baseline subsection | Inline expressions in hotspot-table cells: couples parsing to table layout and can't satisfy "one fenced block per finding" |
| Comment-standard home | Root `AGENTS.md` section | Standalone `docs/COMMENT_STANDARD.md`: adds a document concept + reference hop for one section; agents already load AGENTS.md every session (P1/P8 — fewer homes) |
| Terminal-rule enforcement | Prose + new required manifest phrase (`session-terminal`) in the 5 skills' `required` lists | Prose-only: the drift gate would verify nothing about the new rule; a future edit could silently drop it |
