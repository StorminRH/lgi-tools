## Context

v3.8 proved the resolver model, but three soft spots remain in
`.agent-local/resolve_development_state.py`: terminal-status detection is a
substring match (`INCOMPLETE` currently reads as terminal — confirmed live at
`resolve_development_state.py:33-35`), the 3.9.1.1 marker vocabularies
(`Execution status`, `Baseline effect`, `UX gate`) are specified in the docs but
never validated mechanically, and the resolver is blind to git state. This
session hardens validation and adds opt-in git awareness **without changing the
directive contract** (stages, handlers, seven fields — all frozen this version).

## 1. Goal

The resolver rejects vocabulary/marker violations exactly (naming file and
value), routes missing 3.9-era markers to re-planning, carries a contract's
`UX gate: Yes` into the `session-ready` pause, and — only under a new `--git`
flag — emits advisory warnings for branch/worktree/main-sync drift, while
default output stays byte-identical to today.

**a. `MARKER_VOCABULARY` (module constant)**

```python
# Closed marker vocabularies binding 3.9-and-later artifacts
# (schema owner: docs/SESSION_CONTRACTS.md). Values are exact,
# case-sensitive tokens; anything else is reported with file+value.
MARKER_VOCABULARY = {

    "Baseline effect": ("Improves", "Neutral", "Temporary pressure"),
    "UX gate": ("Yes", "No"),

```
- Decision owned/hidden (P2): the one home for what marker values are legal.
- Callers: session-plan validation, contract validation. No speculative
  entries — only the three markers 3.9.1.1 specified.

**b. `vocabulary_binds(version) -> bool`**
```python
def vocabulary_binds(version: str) -> bool:
    """True when the active version's artifacts must satisfy the 3.9 marker
    vocabulary and presence rules. Pre-3.9 versions are legacy: every new
    validation is skipped for them (never retro-edited, never flagged)."""
```
- Hides the legacy-exemption boundary (one comparison, one owner).

**c. `marker_value_error(path, root, label, value) -> str | None`**
```python
def marker_value_error(path: Path, root: Path, label: str, value: str) -> str | None:
    """Return an errors[] entry naming the offending file and value when a
    PRESENT marker value falls outside its closed vocabulary; None when
    valid. Absent markers are deliberately not this function's concern —
    absence routes to a stage-level re-planning reason, not an error."""
```

**d. `RoadmapRow.terminal` (changed property)**
```python
@property
def terminal(self) -> bool:
    """Exact membership in the closed terminal set after the existing
    cell strip; substring hits (INCOMPLETE, NOT SHIPPED, 'SHIPPED (PR #1)')
    are nonterminal — ambiguous_status() surfaces them as errors."""
```

**e. `ambiguous_status(status) -> bool`**
```python
def ambiguous_status(status: str) -> bool:
    """True when a nonterminal status cell embeds a terminal token
    (e.g. INCOMPLETE, NOT SHIPPED, 'SHIPPED (PR #247)'): the author either
    intended a terminal state or broke the exact-token format. Reported as
    an error with file and value; never treated as terminal."""
```

**f. `git_warnings(root, state) -> list[str]`**
```python
def git_warnings(root: Path, state: dict[str, object]) -> list[str]:
    """Advisory git-snapshot checks behind --git; warnings, never errors,
    never a stage change. Checks: (1) execute-mode directives expect a
    branch embedding the sub-version (pattern below; codex/* exempt; being
    on main warns to create the branch); (2) plan-mode directives warn on a
    dirty worktree; (3) local main behind origin/main (existing refs only —
    no fetch). Every check degrades to silence when git, the repo, or the
    needed refs are unavailable (fixture roots are plain directories)."""
```
- Pulled complexity downward (P5): non-git roots, missing origin, detached
  HEAD all absorbed inside; callers see only strings.
- Uses stdlib `subprocess` (arc constraint: stdlib only).

**g. `approved_session_plan(...)` (changed)** — gains, for binding versions:
missing `Baseline effect` → `(False, "The session plan is missing its
Baseline effect marker.")`; out-of-vocabulary present values → errors[] and
not-ready. Docstring updated to state the split: absence = reason,
bad value = error + not-ready.

**h. UX-gate plumbing (changed `resolve_state` + `directive_for`)** — state
gains `uxGate` (`"Yes"`/`"No"`) parsed from the selected contract's header

`"UX gate: Ryan's local browser review is required before the PR opens; also
pause on any material scope/design conflict."` — otherwise unchanged.
Directive field set untouched.

## 6. Design alternatives (design-it-twice)

**Marker validation placement.**
- *Selected:* one vocabulary constant + tiny `marker_value_error` helper,
  invoked at the existing artifact-read seams (`approved_session_plan`,
  contract selection). Callers learn zero new concepts; the vocabulary has one
  owner.
- *Rejected:* a separate `validate_artifacts()` pass walking all documents
  up front. Rejected because it creates a second traversal that must mirror
  the resolver's active-version/contract-selection logic (P2 leak — two
  owners of "which artifacts matter"), and reports on artifacts the stage
  logic would never read.

**Git awareness integration.**
- *Selected:* `git_warnings()` called only from `main()` when `--git` is set;
  `resolve()`/`resolve_state()` stay pure. Byte-identical default behavior is
  guaranteed structurally (the pure path cannot see the flag), and fixture
  tests of stage logic never touch git.
- *Rejected:* threading a `git: bool` parameter through `resolve_state`.
  Rejected because it widens the pure core's signature for an advisory
  side-channel and makes byte-identical-by-default a property to test rather
  than a property of the structure.

**Branch-pattern expectation (contract decision #2) — resolution:**
warn (execute-mode only) unless branch matches
`^[a-z][a-z0-9-]*/<sub-version>(\.\d+)?-[a-z0-9-]+$` (any lowercase type
prefix — history shows `chore/`, `arch/`, `feat/` — plus the exact
sub-version and a slug), with named exceptions `codex/*` (tooling) and
`main` (which instead warns "create the sub-version branch"). Warn-only, so a
permissive pattern is the right risk posture.

**Marker-failure surfacing (contract decision #1) — resolution:**
- *Missing* 3.9-era marker → stage-level reason (`session-plan-needed`), as
  the contract's Done condition itself specifies.
- *Present but out-of-vocabulary* value → `errors[]` entry naming file and
  value (contract Done condition 1), and the affected plan is additionally
  not-ready. Ambiguous roadmap status cells → `errors[]` naming file and value.

   `UX gate: Yes` contract → pause carries the gate; `No` → unchanged pause.
5. **`--git` warnings:** add `git_warnings` + `--git` argparse flag; payload
   gains a `warnings` key **only** under the flag; `--check` prints warnings
   without failing. Fixtures: `git init` temp repos for (a) dirty worktree
   during a plan directive, (b) wrong-branch during session-ready,
   (c) non-git root → silent; plus a byte-identity test asserting the
   default payload has no `warnings` key and equals the pre-flag output.
6. **Docs + manifest:** update the resolver-behavior sentences in
   `docs/DEVELOPMENT_LIFECYCLE.md` (§1 directive notes + §3/§4: the resolver
