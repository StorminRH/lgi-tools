## 4. Interface designs

### 4a. Runner — `docs/ux-check/run-probes.mjs` (new, ~300 LOC, single deep module)

CLI: `node docs/ux-check/run-probes.mjs [names…] [--list] [--base-url=URL]` — no names
runs every definition in `docs/ux-check/probes/`; per-probe isolation (one crash never
aborts the run); exit non-zero iff any check or default gate failed.

Owns (P5 — callers never see): server-readiness poll, browser/context-per-viewport
lifecycle, the **one** standardized CSP init-script shape, console/pageerror/request
collectors, the Convex/HMR/Speed-Insights noise policy, settle policy, screenshot
pathing, `report.json`, exit aggregation. Writes to `captures/probes/` and wipes **only**
that subdir — the sweep's shots survive, retiring the README's "runs overwrite each
other" caveat.

Header comment (draft):

```js
// run-probes.mjs — the one probe runner for docs/ux-check.
// Owns browser lifecycle, console/pageerror/CSP/network collection, noise
// policy, screenshots, report.json, and the pass/fail exit gate. A probe is a
// definition module in docs/ux-check/probes/ (see README for the format);
// definitions receive everything through ctx and import nothing from here.
// Default gates on every probe: zero style-src CSP violations and zero
// unfiltered console/page errors. Never waits on networkidle: the Convex
// websocket keeps the network busy forever.
```

### 4b. Definition format (contract §7 decision 1)

A tiny ESM module per probe, default-exporting a declarative envelope plus one imperative
`run` hook. Definitions **import nothing** — the runner glob-loads them and injects
everything via `ctx`, so definitions carry zero path coupling:

```js
// docs/ux-check/probes/feedback-dialog.mjs
export default {
  name: 'feedback-dialog',
  route: '/',
  viewports: ['desktop', 'mobile'],   // optional, default both
  settle: 1200,                        // optional ms, default 1000
  allowConsole: [/expected-noise/],    // optional extra noise filters
  async setup({ page }) {},            // optional, pre-navigation (page.route mocks, init scripts)
  async run({ page, viewport, check, shot }) {
    // interactions + assertions
    await check('dialog opens', await page.getByRole('dialog').isVisible());
    await shot('open');
  },
};
```

`ctx` surface (the module's complete public model — P1): `page` (raw Playwright page),
`viewport` (`'desktop' | 'mobile'`), `baseUrl`, `check(label, cond)` (records pass/fail
without throwing), `shot(tag)` (screenshot to
`captures/probes/<name>--<viewport>--<tag>.png`). CSP/console gates are automatic — a
definition asserts only what is specific to it.

- P2: settle policy, noise filters, CSP event shape, capture pathing each get one owner
  (today: four CSP shapes, per-script filter regexes).
- P4: `setup` exists because two real definitions need pre-goto route mocks
  (asset-ring-mock, me-planner); nothing else speculative.
- P7: format documented once in `docs/ux-check/README.md`; skills point at it.

### 4c. Drift-gate check — `check_probe_layout` in `.agent-local/check_agent_drift.py` (new)

Manifest config (new key):

```json
"probeLayout": {
  "definitionsDir": "docs/ux-check/probes",
  "runner": "docs/ux-check/run-probes.mjs",
  "strayPattern": "*-probe.mjs"
}
```

Walks the repo (pruning `.git`, `node_modules`, `.next`, `graphify-out`,
`docs/ux-check/captures`) and reports every `*-probe.mjs` outside `definitionsDir` as a
**warning**: `stray probe script (scratch allowed; delete at close-out): <path>`.
Warning severity is deliberate — the contract keeps scratch scripts allowed mid-session;
close-out reads drift warnings and requires deletion. `main()` gains a local warnings
list feeding the existing `warn:` printer. Runner path joins `requiredPaths` (fails
closed if the runner disappears). Definitions inside `probes/` need no `-probe` suffix —
the directory conveys it — so they can never trip the stray pattern (`run-probes.mjs`
doesn't match `*-probe.mjs` either).

Docstring (draft): `"""Flag stray *-probe.mjs scripts outside the probe-definitions
layout. Warnings, not errors: scratch probes are allowed during a session and deleted at
close-out (contract 3.9.2.5 rail)."""`

## 5. Design alternatives (design-it-twice)

- **Definition format — chosen:** JS module with declarative envelope + imperative
  `run(ctx)`. **Rejected:** pure-data definitions (JSON/YAML action-and-assertion DSL) —
  the real assertion space (clipboard reads, geometry math, mocked ownership payloads,
  `history.replaceState` checks) would force a mini-language wider than Playwright's own
  API; a DSL exposes more concepts than it hides (P1/P4 violation).
- **Runner shape — chosen:** one runner file, ctx-injection, zero imports in definitions.
  **Rejected:** a `runner/` package (browser/collectors/report modules) with definitions
  importing helpers — more files, a public import surface for definitions to couple to,
  and no caller-visible gain; the runner is one cohesive responsibility (P1 corollary:
  length is not the enemy).
