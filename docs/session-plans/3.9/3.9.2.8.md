## 5. Interface designs

**One new export** on the existing leaf (Expand — no new module):

```ts
// src/lib/esi-datasets/freshness.ts

/**
 * The serialized-clock face of the stale-after boundary verdict: true when a
 * row's persisted stale-after instant (epoch ms) has been reached at `nowMs`.
 * Owns the same inclusive `staleAfter <= now` comparison as `isBoundaryStale`,
 * which delegates here. Row existence stays the caller's question — a missing
 * row is an existence verdict, not a boundary one.
 */
export function isBoundaryStaleMs(staleAfterMs: number, nowMs: number): boolean {
  return staleAfterMs <= nowMs;
}
```

`isBoundaryStale` keeps its signature and comment; its body delegates:
`staleAfter === undefined || isBoundaryStaleMs(staleAfter.getTime(), now.getTime())`.
One decision (the inclusive boundary), one owner, two faces for the two row
shapes that exist in the codebase (hydrated Date rows server-side, serialized
epoch-ms rows in the planner's client payloads).

- **Decision owned/hidden (P2):** the exact boundary semantics — inclusive
  `<=`, "reached means stale". Callers no longer encode the comparison.
- **Callers now (P4):** `priceConfidence`, `classifyInput` (both in
  `industry-styles.ts`), plus `isBoundaryStale` itself. Three real call
  sites; no speculative parameters.
- **Edge cases (P5):** null (no price row) deliberately stays planner-side —
  it answers "does a row exist", which drives the `unknown`/`low` levels
  before staleness is ever asked. TS narrowing already guarantees non-null at
  both call sites; the primitive takes plain `number` and cannot receive
  null.
- **Planner policy stays feature-owned:** confidence levels, reason strings,
  share bands, and aggregation remain untouched in `industry-styles.ts`.

## 6. Design alternatives

- **Selected:** ms-shaped second face on the freshness leaf; planner imports
  `isBoundaryStaleMs`. One new concept; zero per-call-site conversion.
- **Rejected — consume `isBoundaryStale` directly:** both call sites would
  each own a null→undefined mapping plus two `Date` allocations per row per
  render pass; more caller knowledge (P1) and it re-introduces the
  Date-construction hazard class at exactly the sites we're cleaning.
- **Rejected — separate zero-import `boundary.ts` leaf (bundle purity):**
  splits one decision across two files to serve a hypothetical bundle-weight
  concern; the leaf chain is pure data (~4 KB, tree-shakeable, public-repo
  content). Speculative (P4).
