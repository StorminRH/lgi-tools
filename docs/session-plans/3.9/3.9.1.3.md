## 5. Interface designs

All new code is Python; the TSDoc-lite standard targets TS exports, so
interfaces here carry module/function docstrings (drafted below). Every
checker exposes the same two-symbol surface: `collect_findings(root) ->
list[Finding]` (in-process, for tests) and `main()` (CLI via the shared
scaffold).

### 5a. `.agent-local/checker_common.py` (new — the one new shared module)

Owns one decision: how a checker finding is represented and delivered.

```python
"""Shared delivery contract for the .agent-local lifecycle checkers.

Owns one decision: how a checker finding is represented and reported.
A Finding is (path, line, message, severity); severity is 'error'
(contradiction; blocks) or 'warn' (suspicious/snapshot-timing; never
blocks). Output is JSON {"errors": [...], "warnings": [...]}; --pretty
indents it; --check exits nonzero only when errors are present.
Markdown marker/table parsing is deliberately NOT here — the resolver
(resolve_development_state.py) owns it; import from there (P2).
"""

@dataclass(frozen=True)
class Finding:
    """One checker verdict anchored to a file and 1-based line."""
    path: str      # repo-relative
    line: int      # 1-based; anchors the finding for the reader
    message: str
    severity: str  # 'error' | 'warn'

def find_line(path: Path, needle: str) -> int:
    """First 1-based line containing needle, or 1 if absent — anchors
    findings whose value came from a line-agnostic resolver helper."""

def run_checker(collect, argv=None) -> int:
    """CLI scaffold: parses --root/--check/--pretty, runs
    collect(root) -> list[Finding], prints the JSON report, and returns
    the exit code (--check: nonzero iff any error-severity finding)."""
```

### 5c. `.agent-local/check_doc_refs.py` (new)

```python
"""Verifies every backticked repo path in docs/, the agent guides
(AGENTS.md, src/AGENTS.md, CLAUDE.md, src/CLAUDE.md), and both skill
trees resolves to an existing file/dir. A backtick span is a path
claim only if it contains '/' and starts with a known repo root
(src/, docs/, scripts/, content/, convex/, public/, fallow-baselines/,
.agent-local/, .agents/, .claude/) or is a known root file
(.env.example, .fallowrc.json, package.json, vercel.json, AGENTS.md,
CLAUDE.md, ...); ':NN'/':NN-MM' line suffixes are stripped; globs must
match >= 1 file; '../' spans are declared archive refs (warn if
missing, since they are operator-machine state); spans with spaces
(commands) and 'dup:'/'zone:'/'sha256:' ids never parse as paths.
ALLOWLIST below carries deliberate legacy references as
(file-glob, token, reason) triples — the allowlist format 3.9.1.1
left to this session."""
```

Severity: dead repo path = **error**; missing `../` archive ref = **warn**.
Basename-only refs (no `/`) are deliberately out of v1 scope (§11).

### 5d. `.agent-local/check_env_example.py` (new)

```python
"""Diffs .env.example against the typed registry in src/lib/env.ts.
Registry keys are parsed from the REQUIRED_ENV / VERBATIM_ENV
'as const' blocks (lines shaped 'KEY: required,'/'KEY: verbatim,';
'//' lines skipped; missing block openers are themselves an error so
an env.ts reshape cannot silently blind the check). An example key is
'present' whether live ('KEY=') or documented-commented ('# KEY=').
Errors: registry key absent from the example (unless in
PLATFORM_INJECTED: VERCEL_ENV, VERCEL_URL, NEXT_RUNTIME) and example
key absent from the registry (unless in NEXT_PUBLIC_ALLOWLIST:
NEXT_PUBLIC_CONVEX_URL, NEXT_PUBLIC_SITE_URL — or TOOLING_KEYS:
CONVEX_DEPLOYMENT, the Convex-CLI var). All findings are errors per
the contract."""
```

To make the live run clean, `.env.example` gains three commented,
one-line-explained entries: `# DOTENV_PATH=`, `# GOOGLE_SITE_VERIFICATION=`,
`# LGI_FORCE_TREE_REBUILD=`. The commented Upstash pair then already counts as
present. Platform-injected keys are allowlisted, never added to the example.

## 6. Design alternatives (design-it-twice records)

1. **Shared markdown parsing** — *chosen:* checkers import
   `marker`/`table_field`/`parse_status_rows`/`parse_audit_findings`/`resolve`
   from `resolve_development_state` (the existing owner; import-safety proven
   by its test suite). *Rejected:* extracting a new `lifecycle_markers.py` —
   churns the just-hardened resolver and its 24 fixtures without reducing what
   any caller must know.
2. **Checker integration** — *chosen:* three standalone scripts with
   `--check`/`--pretty` (arc-mandated) run by the drift harness as
   subprocesses parsing JSON (the `check_tooling` precedent). *Rejected:*
   in-process functions inside `check_agent_drift.py` — no standalone CLI,
   violates the arc constraint, bloats the harness.
3. **Allowlist home** — *chosen:* structured in-script lists with a mandatory
   reason string per entry (consistent with 3.9.1.5's proposed in-script
   pattern list). *Rejected:* a shared allowlist data file — a new format with
   no second consumer (P4) and one more artifact to drift.

## 7. Contract decisions resolved

- **Error vs warn:** intra-state contradictions (AF disagreement,
  trigger/Watch asymmetry, `Complete`-plan-vs-open-roadmap, dead repo path,
  env diff) are **errors**; cross-artifact timing/snapshot cases (SCRATCHPAD
  `Now` lag, delivered-evidence-with-`Pending`, missing operator-machine
  archive ref) are **warns** — matching the contract's proposal, with the
  snapshot-vs-drift case explicitly warn per acceptance criteria.
- **Shared parser home:** existing markdown marker/table parsing stays owned
  by `resolve_development_state.py` and is imported; the new shared surface
  (Finding shape, severity, CLI, `find_line`) lives in `checker_common.py`.
  Three scripts, zero duplicated parsers (P2); 3.9.1.4's declared dependency
  on "3.9.1.3 shared parsing helpers" is satisfied by these two homes.
