## Context

Public-facing documents have drifted, and `/legal` is structurally doomed to keep
drifting because it *enumerates*. An adversarial review (gpt-5.6-sol, high) found
that simply "generalizing" the copy would have **preserved two material privacy
untruths** already live on the page. All findings below are verified against live
code.

**Verified drift:**
- `/legal` claims *"we request exactly four read-only scopes"* + lists them; the
  app requests **12** scopes (`EVE_SCOPES`, `src/features/auth/eve-sso.ts:70-112`,
  test-pinned read-only in `eve-sso.test.ts`). "Exactly four" is false.
- **IP untruth:** `/legal` claims *"We do not record your IP address."* But public
  routes derive the client IP (`clientIdentifier`, `src/lib/rate-limit.ts:82-95`)
  and the rate limiter runs `analytics: true` (`:10-34`), so IP-derived data is
  processed and written. The absolute no-IP claim is not defensible.
- **Deletion untruth:** `/legal` claims data is deleted *"when you unlink"* and
  *"wiped automatically if a character changes hands."* Live code: unlink only
  detaches the account; deletion is the separate **Purge** action; and
  owner-transfer **deliberately retains** the tracker caches — only the credential
  tier is purged (`src/features/auth/owner-transfer.ts:147-149`). Both claims
  overstate deletion.
- **Telemetry breadth:** page-view telemetry stores the raw query string plus
  visitor id / referrer / campaign tags (`src/components/telemetry/page-view-metadata.ts:63`,
  `TelemetryReporter.tsx`), so "which pages, how you arrived" underdescribes.
- `README.md` + `.env.example` re-list the stale four scopes.
- README's tool catalogue omits 3 live tools and advertises an absent
  "coming soon" tool (`README.md:7-14` vs `src/data/tools/registry.ts` `TOOLS`:
  Wormhole Sites, Industry Planner, Skill Queues, Industry Jobs, Structures — no
  Roll Calculator).
- README + CONTRIBUTING describe skills/jobs as **Convex-backed**; they moved to
  **Neon** in MIGRATE.B and Convex now serves only the online-status canary
  (`convex/schema.ts:9-15`). CONTRIBUTING's "every route prerenders a static
  shell" also overstates (dynamic routes exist in `scripts/route-classification.json`).
- `.env.example` labels `CONVEX_SERVICE_SECRET`/`SITE_URL` "future / not consumed
  yet" (`:140-142`) though both are consumed now.
- `.github/PULL_REQUEST_TEMPLATE.md` uses a `## Summary` format the canonical
  spec forbids (`docs/PR_REVIEW.md:31-41`).
