## Design alternatives

1. **Lint integration — chosen:** a dedicated scoped block (jsdoc/tsdoc rules
   never interact with the existing `no-restricted-syntax` replace-semantics
   blocks). *Rejected:* folding the rules into the base `**/*.{ts,tsx}` block —
   it would lint tests and root configs and force carve-outs the contract
   forbids.
2. **Authoring order — chosen:** author all batches first, flip the rules
   last; every commit keeps `pnpm verify` green. *Rejected:* flip-first with
   the tree red for the whole session — breaks the always-green rule and makes
   partial commits unshippable.
3. **Batching — chosen:** eight by-slice batches, each closed by a scoped
   presence-rule re-run against the scratch config, one commit each —
   reviewable diffs with coherent domain context. *Rejected:* by-export-kind
   batching (types, then functions…) — scatters the reading context needed to
   write truthful contracts across every file three times.
