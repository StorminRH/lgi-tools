## Context

The rule ("Convex only for ≤2-min-cache live data; everything slower → Neon") lives in agent-guide prose plus SELF_REVIEW §1's manual "look it up at esi.evetech.net/ui/, don't guess" checklist — a human judgment gate with a documented failure (the 3.7.5.1 owned-blueprints mis-placement; skills/jobs were also originally mis-placed in Convex and migrated out in MIGRATE.B). ## 5. Interface designs

### 5.1 The resolved layering (the contract's §7 decision — presented for approval)

The contract's pinned structure says "the entries manifest + CI validator compose above the slices," and flags the by-name-reference question as a decision this plan must resolve. **Resolution: the entries data joins the type in the `src/lib/` leaf; only the checks + gate test compose above the slices.** Three constraints force this jointly:

1. Rail part 2 (session .2) bans TTL/staleness-window declarations *outside the leaf* — so the effective-TTL numbers must be leaf-resident, or .2 must move every entry (violating the contract's "no second pass over the entries").
2. Session .2's verdict reads "the entry's model + effective TTL," and slices may import only the leaf (lib→lib; the purge discipline bans feature→junction value edges) — so the entry data the verdict consumes must be reachable from the leaf.
3. This is legal with zero import violations because **every world-reference in an entry is by-name**: `OwnerSyncDescriptor` has no name/id field (transient factory values — claim-by-object is impossible anyway, resolving the contract's anticipated tension), cron routes are already keyed by URL path string in `src/app/api/cron/registry.test.ts`, deferred-queue datasets have stable handles in `ESI_REFRESH_DATASETS`, and table claims are SQL-name strings. Rename drift is answered by the validator: a renamed table/route/handle stops matching its entry string → red gate (the same fail-closed property claim-by-object buys the purge registry).

The three pinned one-primitive properties hold: one entry per dataset (leaf), a verdict with zero configuration of its own (lands in .2, leaf), and the rail banning staleness computation elsewhere (.2).

### 5.2 New modules

**Leaf — `src/lib/esi-datasets/types.ts` + `entries.ts`** (lib zone; imports nothing outside lib; pure data):

- `EsiDatasetEntry` — a discriminated union on `shape` (`'global-cron' | 'personal-on-view' | 'live'`) so each placement carries exactly its legal freshness models and refresh-owner reference; common fields: `name`, `store` (`'neon' | 'convex'`), `upstream`, `mirrorTables` (SQL-name strings), optional `ttlOverride` (value + required rationale — an override without a rationale is unrepresentable), optional `collaborative`, optional `waiver`, optional `notes`/`cronBackstopRoute`.
- `EsiUpstream` — discriminated: `{ kind: 'esi'; specPaths: readonly string[]; verifiedCacheSeconds: number } | { kind: 'ccp-sde-manifest' } | { kind: 'google-gsc' }`. The ESI cache-time rules (Convex ≤120s, TTL ≥ upstream) apply only to `kind: 'esi'`; the placement rules apply to every entry — which is why SDE and GSC (named by the master plan as lacking any declaration point) belong in the registry despite non-ESI upstreams.
- `EsiGateRuleId` — `'convex-cache-bound' | 'global-cron-names-route' | 'personal-names-owner' | 'ttl-at-least-upstream'`; a waiver names exactly one rule.
- `effectiveTtlMs(entry)` — override wins, else verified upstream cache; `null` for models with no static window (expires-boundary, cron-cadence).
- `ESI_DATASET_ENTRIES` — the 13 entries (table below).

**Freshness-model vocabulary (deliberate extension, approved with this plan):** the contract lists `caller-ttl | row-stale-after | expires-boundary`. Two honest additions: `cron-cadence` (indices/SDE/GSC — no TTL constant exists; vercel.json owns the cadence; forcing `caller-ttl` would fabricate a number) and `engine-cadence` (online status — the sync engine schedules off stored ESI `Expires` floored at `cadenceFloorMs`).

**Junction — `src/esi-datasets/checks.ts` + `registry.test.ts`** (new top-level unzoned junction mirroring `src/purge/`; imports the world; **no Fallow zone/rule change** — being unzoned is what grants the upward value-imports, and zoning it would break that):

- `isEsiMirrorTable(table)` — DB-free `getTableConfig` scan; flags columns matching `/_refreshed_at$/`, exact `stale_after`, or exact `fetched_at`. A fail-closed net for the mirror *pattern*, not a census (mirrors purge's identity-column rule). Verified live: flags the six sync surfaces + `characters` (`affiliation_refreshed_at`) + `market_prices` + `market_history` + `esi_snapshots`.
- `findUnregisteredMirrors(flagged, claimed, infrastructure)` — pure set difference (purge's `findUnclaimed` pattern) so the red path is unit-testable.
- `checkEntries(entries, context)` — the pure rules engine over live-world name sets (vercel.json cron paths, `ESI_REFRESH_DATASETS`, `SYNC_DATASETS`); returns findings; a waived rule reports as waived, never silently skipped.
- `ESI_INFRASTRUCTURE_TABLES` — declared exemptions (`esi_snapshots`: transport-layer response cache, etag/304 + retention-pruned, not a dataset). `esi_refresh_jobs` has no freshness column, is never flagged → comment only, no speculative exemption.
- `CONVEX_ESI_HOMES` — the `NON_NEON_HOMES` precedent: `convex:characterOnline` → entry `online_status`; the gate requires every `store:'convex'` entry to appear here and vice versa.

Draft `/** */` interface comments (P7-first; written before implementation):

```ts
/**
 * The upstream a dataset is fed from. ESI upstreams carry the hand-verified
 * cache time and the spec path(s) it was read from at esi.evetech.net/ui/ —
 * the citation, since no vendored spec exists. Non-ESI upstreams (CCP's SDE
 * manifest, Google Search Console) are exempt from the ESI cache-time rules
 * but not from the placement rules.
 */
export type EsiUpstream = …

/**
 * The rules an entry may carry a waiver against. A waiver names exactly one
 * rule — recorded debt, never a blanket blessing.
 */
export type EsiGateRuleId = …

/**
 * One declaration per ESI-fed dataset: where it lives, how fresh it must be,
 * and who refreshes it. The shape discriminant pairs each placement with its
 * legal freshness models and refresh-owner reference; all world references
 * are by name (table SQL names, cron route paths, ESI_REFRESH_DATASETS
 * handles, sync-engine dataset keys) so this leaf imports nothing — the
 * junction validator (src/esi-datasets/) cross-checks every name against the
 * live world. Adding a dataset means writing one entry here and letting the
 * gate arbitrate it.
 */
export type EsiDatasetEntry = …

/**
 * The dataset's effective staleness window in ms: the declared override when
 * one exists (with its recorded rationale), otherwise the verified upstream
 * cache time — "never poll faster than upstream" as the default, not a
 * comment. Returns null for models with no static window (expires-boundary,
 * cron-cadence), where the row or the cron schedule owns freshness.
 */
export function effectiveTtlMs(entry: EsiDatasetEntry): number | null

/**
 * THE registry: exactly one entry per ESI-fed dataset, complete from day one
 * (freshness fields included). This is the single owner of the placement and
 * staleness-window decisions the slice TTL constants currently duplicate;
 * session .2 collapses those constants onto it. Enforced by
 * src/esi-datasets/registry.test.ts — an unregistered dataset, a misplaced
 * store, or a below-upstream TTL is a red gate, not a review comment.
 */
export const ESI_DATASET_ENTRIES: readonly EsiDatasetEntry[]

/**
 * True when a Neon table carries an ESI-mirror freshness column
 * (…_refreshed_at, stale_after, fetched_at) — the set the gate requires a
 * registry entry (or a declared infrastructure exemption) for. DB-free:
 * getTableConfig reads pgTable metadata with no connection. A safety net for
 * the mirror pattern, not a census — tables with only generic timestamps are
 * covered by their entry's mirrorTables, not by this scan.
 */
export function isEsiMirrorTable(table: PgTable): boolean

/**
 * The gate's mirror assertion as a pure set difference so the red path is
 * unit-testable on synthetic input: returns flagged table names neither
 * claimed by any entry nor declared infrastructure.
 */
export function findUnregisteredMirrors(
  flagged: readonly string[],
  claimed: ReadonlySet<string>,
  infrastructure: ReadonlySet<string>,
): string[]

/**
 * The placement/freshness rules engine, pure over its inputs: entries plus a
 * context of live-world name sets (vercel.json cron paths, the deferred-queue
 * dataset handles, the sync-engine dataset keys). Returns human-readable
 * findings; empty means green. Rules: convex requires verified cache ≤120s or
 * a collaborative flag; global-cron names a live cron route or carries a
 * waiver; personal-on-view names a real deferred-queue dataset or entry
 * point; effective TTL ≥ verified upstream cache; every waiver rationale
 * non-empty. A rule an entry waivers is reported as waived, never silently
 * skipped-and-forgotten.
 */
export function checkEntries(entries, context): string[]

/**
 * Durable Neon tables the mirror scan flags that are ESI *transport*
 * infrastructure, not datasets (esi_snapshots: response-body cache governed
 * by etag/304 and retention pruning). The declared-exemption discipline of
 * purge's NON_NEON_HOMES: explicit and audited, never a silent omission.
 */
export const ESI_INFRASTRUCTURE_TABLES: readonly { table: string; reason: string }[]

/**
 * ESI-fed data homes outside Neon, invisible to schema reflection, each
 * mapped to the registry entry that owns it. The gate pins this list and
 * requires every store:'convex' entry to appear here.
 */
export const CONVEX_ESI_HOMES: readonly { home: string; entry: string }[]
```

## 6. Design alternatives (design-it-twice records)

- **TTL source-of-truth direction:** *Selected* — entries declare literal numbers; the gate test cross-checks them against the live slice constants as characterization (block commented ".2 deletes these constants and these assertions with them"). Zero-drift in .1 (divergence = red test), and the registry is already the independent declaration .2 collapses onto. *Rejected* — manifest imports the slice constants as values: makes the registry downstream of the slices, exactly backwards from .2's end state.
- **Junction location:** *Selected* — new top-level `src/esi-datasets/` mirroring `src/purge/`. *Rejected* — under `src/db/`: this registry governs placement across Neon *and* Convex; parking it in `db/` misstates its scope.
- **Entry shape:** *Selected* — discriminated union on `shape`; illegal owner/model pairings unrepresentable (P5), while every gate-checked rule remains violable within the real type (seeded-red tests need representable violations — no widened test-only input type). *Rejected* — flat optional fields: every entry becomes a puzzle of which optionals apply, and nonsense combinations become runtime findings the gate must enumerate.
- **Market-history conformance:** *Selected* — `global-cron` + `cronRoute: null` + waiver (recorded debt). *Rejected* — a fourth shape `'global-on-view'`: blesses the placement silently and widens the pinned enum for one dataset.
- **Convex coverage:** *Selected* — declared `CONVEX_ESI_HOMES` list (purge's `NON_NEON_HOMES` precedent), pinned by the gate. *Rejected* — reflecting `convex/schema.ts` in the test: imports Convex server machinery into vitest for one table.

## 8. Test and verification plan

**Behavioral tests (pure functions):** `isEsiMirrorTable` (suffix rule hits `last_refreshed_at` and `affiliation_refreshed_at`; `updated_at` alone → false); `findUnregisteredMirrors` (seeded unregistered name surfaces; claimed/infrastructure clear); `effectiveTtlMs` (default = upstream; override wins; null for expires-boundary/cron-cadence).

**Seeded-red classes (contract §8 — one violation, one finding, synthetic entries):**
1. Bad placement: `store:'convex'`, upstream 3600s, no collaborative flag, no waiver.
2. Cron route not in the live cron-path set; and `cronRoute: null` without waiver.
3. Personal owner naming a dataset outside `ESI_REFRESH_DATASETS`.
4. Below-upstream TTL: override 60s vs upstream 300s.
5. Unregistered mirror table.
6. Waiver mechanics: class-2 violation with rationale-bearing waiver → no finding; same waiver with empty rationale → finding.

**Live-green assertions:** `checkEntries(ESI_DATASET_ENTRIES, liveContext)` → `[]` (context from `vercel.json` + `ESI_REFRESH_DATASETS` + `SYNC_DATASETS`); frozen sorted snapshot of reflected flagged tables (scan-regression guard both directions); every flagged table claimed-or-infrastructure; stale-claim check (every claimed name exists in the schema); entry names unique; every `ESI_REFRESH_DATASETS` member claimed by exactly one entry; `CONVEX_ESI_HOMES` ↔ convex-store entries bidirectional; the market-history waiver present with non-empty rationale (pins the one live waiver).

**Characterization cross-checks (deleted in .2 with the constants):** entry effective TTLs equal `SKILLS_TTL_MS`, `JOBS_TTL_MS` (both job entries), `ASSETS_TTL_MS`, `BLUEPRINTS_TTL_MS`, `STRUCTURES_TTL_MS`, `AFFILIATION_TTL_MS`, `STALE_AFTER_TTL_MS`, `SYNC_DATASET_CONFIG.onlineStatus.cadenceFloorMs`.
