## Goal

A cron route becomes a declaration handed to one shell primitive that owns
auth → (pre-lock gate) → advisory lock → work → telemetry ordering, wake-class
declaration, and the noteworthy-gated recording policy; the five batch routes
and the sweeper migrate with behavior preserved, and two rails (schedule
cross-check gate, restricted-import lint) enforce the model.

## Context

Seven Vercel cron routes each re-assemble the same wiring — logger creation,
start-time capture, busy/refreshed response construction, lock-key plumbing,
telemetry emission — around the shared gate (`runCronJob` / `requireCronAuth`
/ `cronLogger`), including whole-version clone `dup:b54bf337` (AF-009 Watch).
The driver is not the clone but the PR #159 idle-wake regression: the "healthy
no-op must not touch Neon" decision was encoded as a route-local predicate
(`isNoteworthySweep`), so the next 15-minute cron (`drain-esi-refresh-jobs`)
rebuilt the failure mode three-fold (~96 Neon wakes/day idle). This session
makes a cron route a *declaration* — name, wake class, lock policy, recording
policy, work — with one shell primitive owning the ordering guarantee, and
lands the schedule cross-check gate + restricted-import rail so the decision
can never silently regress again. Session .2 (same branch) adds the drain's
Redis idle probe.

## Interface designs (with draft interface comments)

All new exports live in the expanded `src/db/cron-gate.ts` (it already binds
`directClient` to the lock scaffold; `src/db` composition may import
`data/telemetry`). `runCronJob` remains exported through this session solely
as the drain's path (deleted in .2 — full pattern closes at branch close).

```ts
/**
 * Wake class a cron declares: `batch` for daily jobs whose purpose is waking
 * Neon; `idle-silent` for sub-daily watchdogs/drains whose healthy no-op run
 * must touch zero Neon (no lock, no read, no write). The schedule cross-check
 * gate requires every sub-daily schedule to map to an `idle-silent`
 * declaration or carry a written justification.
 */
export type CronWakeClass = 'batch' | 'idle-silent';

/**
 * Handles the shell passes to a declaration's stages so route files never
 * import DB or telemetry primitives directly: the shared direct Neon client,
 * the reserved lock connection when the declaration is lock-guarded, and
 * `record` for route-specific durable telemetry events (swallowed; runs with
 * Neon already awake).
 */
export type CronWorkContext = {
  client: Sql;                       // the shared directClient
  reserved?: ReservedConnection;     // present iff the declaration locks
  record: (action: UsageAction, metadata: Record<string, unknown>) => Promise<void>;
};

/**
 * What one cron run produced. `outcome` labels the boundary log line and the
 * durable telemetry row; `workDone` feeds the recording policy (under
 * `noteworthy`, a durable row is written only when work was done or the run
 * failed); `telemetry` is the durable/boundary metadata (the shell appends
 * durationMs); `body` is the route's typed JSON response, returned verbatim.
 */
export type CronRunOutcome<Body> = {
  outcome: string;
  workDone: boolean;
  telemetry?: Record<string, unknown>;
  body: Body;
};

/**
 * One cron route as a declaration: identity (telemetry scope + action), wake
 * class, lock policy, recording policy, optional pre-lock gate, and the work.
 * `defineCronRoute` is the only way a route reaches auth, the advisory lock,
 * or durable cron telemetry (lint-enforced), so the ordering guarantee —
 * auth → preLock → advisory lock → work → telemetry — and the recording
 * policy cannot be bypassed or reordered. Lock contention short-circuits to
 * the declared busy body; `record: 'always'` requires a written justification
 * (the default records only failure or work-done). `preLock` runs before any
 * lock is reserved and either short-circuits with a full outcome or hands a
 * typed value to `work` (the SDE version gate's shape).
 */
export type CronRouteDeclaration<Body, Pre = void> = {
  name: string;                       // boundary-line/telemetry scope, e.g. 'cron:affiliations'
  action: UsageAction;                // existing vocabulary; no new actions
  wakeClass: CronWakeClass;
  record:
    | { policy: 'noteworthy' }
    | { policy: 'always'; justification: string };
  lock:
    | { key: number; busyBody: (durationMs: number) => Body }
    | { mode: 'none'; justification: string };
  preLock?: (ctx: CronWorkContext) =>
    Promise<{ done: CronRunOutcome<Body> } | { proceed: Pre }>;
  work: (ctx: CronWorkContext, pre: Pre) => Promise<CronRunOutcome<Body>>;
};

/**
 * Builds the GET handler for one declared cron route. Owns bearer auth, the
 * stage ordering, duration capture, the every-run structured boundary line,
 * the recording-policy-gated durable telemetry row, the busy-path response,
 * and failure recording (a thrown `work` is recorded as a failure row, then
 * rethrown so the platform 500 is unchanged). Route files export the result
 * as `GET` and keep only static segment config beside it.
 */
export function defineCronRoute<Body, Pre = void>(
  declaration: CronRouteDeclaration<Body, Pre>,
): (req: Request) => Promise<Response>;
```

Step-3 answers: the shell owns and hides *the ordering + recording decision*
(P2 — the PR #159 lesson gets one home); its surface (one declaration type,
one factory) is far simpler than the composed implementation (P1); it lives in
`src/db` because it binds `directClient` (zone-legal: db → lib, db → data);
every field has a real consumer among the six migrating routes (P4 — the
idle-probe slot is deliberately **not** added until .2's consumer exists);
busy, duration, boundary-line, and failure-recording edge cases are absorbed
inside (P5). Rationale comments carry the #159 story and the "gate, not job
merger" lineage.

Per-route: each route gains a sibling `declaration.ts` (the `noteworthy.ts`
precedent) exporting its typed declaration; `route.ts` shrinks to the doc
comment, static `maxDuration`, markers, and
`export const GET = defineCronRoute(declaration)`. This exists because Next 16
route files can't export arbitrary fields — the schedule gate imports
declarations without touching route modules.

Route-local behavior stays visibly route-local in each declaration's stages:
SDE's pre-lock version gate (`preLock`), GSC's prune sweep + quota handling
(inside `work`), prices' deliberate lock-free mode + degradation telemetry via
`ctx.record` + Discord alert + `revalidateTag`/domain event (inside `work`),
sweeper's Convex sweep + loud re-arm log (inside `work`, `isNoteworthySweep`
maps to `workDone`).

**Recording-policy assignments (behavior-preserving):** the five batch routes
declare `always` with a one-line justification (daily cadence — Neon woken by
design; preserves today's O-2/O-3 no-op rows: prices `skipped`, SDE
`up-to-date`/`remote-unreachable`, busy rows). Sweeper declares `noteworthy`
(exactly today's #159 behavior). Two named deltas, accepted and documented:
(1) GSC's *telemetry* busy label normalizes from `skipped/busy` to the shell's
uniform `busy` (its response body keeps `status:'skipped', reason:'busy'` —
bodies are route-owned); (2) a thrown `work` now writes a failure telemetry
row before rethrowing (response unchanged; the contract's recording policy
names failure as recordable).

## Design alternatives (design-it-twice)

- **Selected:** thin `defineCronRoute(declaration)` factory over the existing
  primitives (Option B). A route's caller-visible model collapses to one
  concept — the declaration — and the shell can own logger creation,
  start-time capture, busy construction, and emission, which an options-grown
  gate never could. Not a P3 pass-through: it presents a genuinely different
  mental model (route = declaration) and owns decisions `runCronJob` doesn't
  (ordering guarantee, recording policy, wake class, duration/boundary line,
  failure recording).
- **Rejected:** growing `runCronJob`'s options (Option A) — each route would
  still assemble its logger, start time, and responses around the call, the
  options bag would widen a gate primitive toward special-case creep (flags
  for lock-free, telemetry, pre-lock), and the clone would survive in the
  assembly that remains. One sentence for the owning comment: *the gate
  stayed a gate; declaring the whole route is what removed the per-route
  assembly.*
