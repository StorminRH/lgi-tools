## Goal

The drift gate derives its hand-synced expectations from their existing
sources of truth — the active version's contract index and the paired-skill
registry — so the per-version manifest update ritual disappears without
weakening any guarantee.

## Context

The agent-drift gate (`.agent-local/check_agent_drift.py`) validates parity
between shared policy and both runtime adapters against
`.agent-local/policy-manifest.json`. Two manifest sections are hand-synced
copies of facts that already have a source of truth, and they impose a
recurring maintenance ritual:

- `sessionContracts.expected` hand-lists all 31 contract paths for v3.9. The
  3.8 archive required manually emptying it; 3.9 contract creation required
  manually repopulating it. The true owner of "which contracts exist for the
  active version" is `docs/session-contracts/<version>/INDEX.md` plus the
  resolver's active-version discovery.
- `requiredPaths` enumerates all 20 `SKILL.md` paths (10 skills × 2 runtimes)
  that `pairedSkills` × `skillRoots` already imply — and `check_skill_pairs`
  already enforces their existence via the set-equality check and per-file
  parse.

The contract also requires a pair-by-pair review of the regex-phrase
requirements: each stays with a one-line note of the drift it guards, or is
replaced by a structural check. Goal: fewer prose-coupled patterns, not zero.

## Design

### D1 — Derive `sessionContracts.expected` at check time (contract's proposed option, confirmed)

Check-time derivation, no generated files. `check_agent_drift.py` imports the
two owning parsers from the resolver module (same directory; same-dir imports
are the established pattern in the `test_*.py` files).

New function (interface comment first, per P7):

```python
def derived_session_contracts(root: Path) -> list[str]:
    """Return the active version's indexed session-contract paths.

    The roadmap discovery and INDEX.md format stay owned by the lifecycle
    resolver; this derives the drift gate's expectations from them instead of
    a hand-synced manifest list. No active master plan or no index yields an
    empty list without error: those lifecycle states are the resolver's to
    report, and the empty set is exactly the legal post-archive boundary
    state the old hand-list had to be manually emptied into.
    """
```

Implementation: `active_roadmap(root)` → version; parse
`docs/session-contracts/<version>/INDEX.md` with `parse_contract_index()`;
return the indexed contract paths as **sorted repo-relative posix strings** —
the exact shape the hand-list had, so `scan_paths = scan + derived`,
`ROOT / raw_path`, the `path.stem` heading identity, and error-message
ordering all behave byte-identically. Roadmap-parse errors are NOT re-reported
here (the gate's `check_development_state` step already surfaces resolver
errors; duplicating them would double-report every finding).

`check_session_contracts` keeps its existing per-contract checks unchanged
(file exists, first heading `## Session <id> `, no stray `# Phase` heading,
forbidden-pattern scan over `scan` + derived paths), and gains one **orphan
check**: any `*.md` directly in the active version's contract folder
(non-recursive) that is neither `INDEX.md` nor indexed is an error. This
preserves the hand-list's real guarantee (the folder and the gate can't drift
apart) in both directions without enumeration — de-indexing a still-present
contract file fails the gate again, as it did under the hand-list.

Manifest: the `sessionContracts.expected` key is deleted; `scan` and
`forbidden` stay.

**Rejected alternative (design-it-twice):** a generated `expected` section
written into the manifest by a new sync command. Rejected because it creates a
second artifact that can drift (the exact failure mode this session removes)
and adds a regeneration ritual; nothing needs the list to exist statically.

### D2 — Derive required skill paths from `pairedSkills` × `skillRoots`

Delete the 20 `SKILL.md` entries from `requiredPaths` and have `check_paths`
extend its iterated list with the derived paths (`skillRoots` ×
`pairedSkills` → `<root>/<skill>/SKILL.md`). Deriving inside `check_paths` —
rather than leaning on `check_skill_pairs`' existing set-mismatch/missing-file
errors alone — preserves the exact "missing required path" error signature for
this violation class, keeping the seeded matrix's before/after output
identical.

Keep explicit in `requiredPaths`:

- the 7 `agents/openai.yaml` entries — genuinely irregular (only 7 of 10 codex
  skills have one); a single-owner hand-list, not duplication;
- the `.agent-local/*.py` checker/test paths — `check_development_state_tests`
  globs `test_*.py`, so a deleted test file would otherwise vanish silently;
  these entries are load-bearing;
- `.claude/settings.json` / `.codex/hooks.json` — technically implied by
  `hookConfigs`, but removing them is not named by the contract; deliberately
  retained duplication (scope guard).

**Rejected alternative:** a `codexAgentConfigs` manifest key to derive the
openai.yaml paths. Rejected per P4 — it adds a schema concept to re-express a
7-line list that appears exactly once.

### D4 — Testability seam + fixtures

`check_agent_drift.py` currently uses module-level `ROOT`/`MANIFEST_PATH` and
has **no test file**. To fixture-test the derivation per the established
pattern (`test_env_example.py`-style tempdir + `unittest`), thread a
`root: Path` parameter through the checks this session touches and their
helpers (`relative`, `read_text`, `parse_skill`, `check_paths`,
`check_session_contracts`, `check_skill_pairs`), resolver-style, with `main()`
passing the current `ROOT`. (This mirrors the repo's only established test
seam — `resolve(root)`, `collect_findings(root)`; no existing test
monkeypatches globals.) Behavior-preserving (P9); the subprocess-based checks
(`check_ignored`, `check_development_state`, `check_development_state_tests`,
`check_lifecycle_checkers`, `check_tooling`) are untouched this session and
keep the global.

New `test_agent_drift.py` (auto-discovered by the gate's own `test_*.py`
glob — it will run inside every future drift check, and is itself added to
`requiredPaths` for convention parity with every other checker's test file):
tempdir fixtures covering, for the affected checks, both the passing state and
each seeded violation class:

1. missing canonical guide / required path (`check_paths`)
2. retired path exists (`check_paths`)
3. skill set mismatch — missing and extra skill dir (`check_skill_pairs`,
   proves D2 lost nothing)
4. missing SKILL.md revision marker / required phrase / forbidden phrase /
   wrong-runtime phrase (`check_skill_pairs`)
5. derived contracts: indexed file missing; wrong first heading; stray
   `# Phase` heading; forbidden contract policy present
   (`check_session_contracts`)
6. orphan contract file not in INDEX (new class)
7. no-active-version and no-INDEX states yield an empty expectation set with
   no error (the archive-boundary behavior)

The remaining violation classes (hooks, imports, ignored paths, resolver
directive shape, checker propagation, tooling parity) are code-untouched; they
are verified by the live seeded-violation matrix below rather than backfilled
fixtures (no coverage padding, P10).

## Ordered implementation steps (each keeps the gate green)

1. **Seam refactor (behavior-preserving):** thread `root` through
   `relative`/`read_text`/`parse_skill`/`check_paths`/`check_skill_pairs`/
   `check_session_contracts`; run the full gate — output identical.
2. **Characterization fixtures:** add `test_agent_drift.py` covering current
   behavior of the three checks against a tempdir fixture manifest that still
   hand-lists contracts.
3. **D1:** add `derived_session_contracts()` + orphan check; switch
   `check_session_contracts` to it; delete `sessionContracts.expected` from
   the manifest; update fixtures to the derived model (INDEX-driven),
   including empty-state and orphan cases.
4. **D2:** extend `check_paths` with the derived skill paths; delete the 20
   `SKILL.md` entries from `requiredPaths`; add the new `test_agent_drift.py`
   fixture file to `requiredPaths`; fixture proves the missing-SKILL.md class
   fails with the identical error signature.
5. **D3:** apply the three approved pattern replacements in `pairedSkills`.
6. **Revision bump:** `python3 .agent-local/bump_policy_revision.py 20`
   (manifest + all 20 skill markers; `bump_policy_revision.py` itself needs no
   change — it round-trips the manifest and never touches the deleted key).
7. **Seeded-violation matrix (live):** for every violation class the gate
   failed on before (paths, imports, hooks, skill set/frontmatter/marker/
   phrases, ignored paths, contracts, resolver shape, failing test file,
   checker propagation, tooling parity), seed the violation in the working
   tree, run `python3 .agent-local/check_agent_drift.py`, record the failure,
   revert. Record the matrix + the before/after manifest diff.
8. **Housekeeping rider:** delete the stray empty untracked
   `docs/session-contracts/3.9 2/` directory (Finder-duplicate artifact;
   invisible to git/resolver but demonstrates a blind-spot class — a
   duplicated `VERSION_3_9_PLAN 2.md` would be silently ignored by
   `active_roadmap`'s `re.fullmatch`; noted, not fixed here).
9. **Close-out artifacts:** changelog entry in `content/changelog/v3.9.md` +
   `APP_VERSION` → 3.9.1.6; `pnpm verify`; full close-out per the contract
   (pre-pr-design-review → PR → Greptile → merge → production
   reconciliation).
