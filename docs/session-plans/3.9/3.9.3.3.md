invalid site detail (`/sites/100`) renders its otherwise-correct 404 + noindex

## 5. Interface designs

No new modules, hooks, components, or API surfaces. The primary fix is one
route segment config export plus its rationale comment (P5 — the failure mode
is defined out of existence: invalid ids never render, so there is no shell
resume to abort):

```ts
/**
 * The wormhole-site catalogue is a closed set: generateStaticParams
 * prerenders every valid id, so an unknown id is answered by the static 404
 * instead of resuming the PPR fallback shell. That aborted resume is what
 * emitted the production React #419 pair on invalid ids (3.9.3.3); the
 * per-deploy id set matches the existing cacheLife('max') search index.
 */
export const dynamicParams = false;
```

Diagnosis probes (industry discriminator, boundary census) run from the
scratchpad only and are never committed. No speculative exports.

## 6. Design alternatives

- **Selected — (a) `dynamicParams = false`:** with every segment declaring it,
  Next registers `FallbackMode.NOT_FOUND` (compiled
  `build/static-paths/app.js:625–630`) — no fallback shell exists, unknown
  ids get the fully static root `not-found.tsx` (404 + auto noindex), nothing
  streams, no boundary aborts. Smallest public model; one line + comment.
- **Rejected — (b) restructure the page to stream no incomplete boundaries:**
  structurally unreachable — the shell's holes are postponed at build time
  while params arrive at request time; empirically falsified by the
  3.8.4.1.1 revert. The working variant (delete the Suspense hole + streamed
  metadata) regresses the ◐ design and the freshness strip.
- **Fallback — (d) proxy-level rejection of invalid ids:** works
  (`streaming.md:670`) but duplicates catalogue knowledge at the edge (P2
  leak); chosen only if (a) hard-fails at build.
- **Conditional — (c) Next upgrade:** only if the baseline re-reproduction
  shows upstream drift; requires explicit operator approval (scope risk).
- **Rejected outright:** any console/gate suppression of #419.

**Known risk carried into execution:** `dynamicParams.md:22` claims the option
"is not available when Cache Components is enabled," yet the compiled build
code honors it and errors (E280) only on missing params. The P2 preview build
is the deciding experiment; on build rejection, fall to (d).
