## 1. Goal

A stale baseline number, a baseline row naming a deleted file, or a tripped
Watch trigger is detected mechanically at close-out (and on demand) instead of
by the next audit's manual re-measurement. Checkers report; they never mutate
the baseline or auto-promote (P10). Additionally (operator-directed
amendment): the workflow system itself — docs, skills, checkers, manifest — is
public, tracked, and reviewed by the same PR gate as application code, with
local secrets, generated artifacts, and machine state deliberately left
behind.

### 5a. `repo_measures.py` (new in `.agent-local/` — the one shared module)

Owns one decision: **how a countable repo fact is measured** (P2 — both
checkers need exports, zone files, and clone groups; without one owner the
measurement rules would leak into two scripts).

```python
"""How a countable repository fact is measured, for the baseline-facing
checkers. One owner for the counting rules so the claims checker and the
watch-trigger checker cannot drift apart. All counts mirror the audit
commands in docs/VERSION_AUDIT.md Step 1 (src+convex, *.ts/*.tsx,
excluding *.test.* and _generated). Raises MeasureError when a fact
cannot be measured (missing file, unknown/autoDiscover zone, fallow
failure); callers convert that to an error Finding."""

class MeasureError(Exception):
    """A fact could not be measured; message says why."""

def production_file_count(root) -> int: ...
def test_file_count(root) -> int: ...
def production_loc(root) -> int: ...
def suppression_count(root) -> int:
    """eslint-disable | @ts-expect-error | fallow-ignore hits in src+convex."""
def export_count(root, rel_path) -> int:
    """Count of lines starting 'export' in one named repo file.
    MeasureError if the file does not exist."""
def zone_file_count(root, zone_name) -> int:
    """Files assigned to one named .fallowrc.json boundary zone,
    honoring first-match-wins against earlier zones. Explicit-`patterns`
    zones only; an autoDiscover zone (or unknown name) is MeasureError —
    the trigger grammar is a closed spec, not a checker feature."""
def clone_file_counts(root, run_dupes=None) -> dict[str, int]:
    """fingerprint -> distinct instance-file count from
    `fallow dupes --format json`. run_dupes is the subprocess seam
    (tests inject parsed JSON); default runs the repo's fallow binary."""
```

### 5b. `check_watch_triggers.py` (new in `.agent-local/`)

```python
"""Evaluates every fenced watch-trigger block in the health baseline
using the closed grammar owned by docs/VERSION_AUDIT.md Step 4
(trip-form: expression true == trigger fired). A tripped trigger is a
WARN — 'promote AF-NNN' with the measured value — never an exit-code
failure: classification stays an audit decision (P10). Errors are
reserved for contradictions: an unparseable trigger line, or a trigger
whose subject cannot be measured (dead file, unknown zone). A dup:
group absent from the live dupes run measures 0 (the clone dissolved;
trip-form handles it). Measurement rules live in repo_measures."""
```

Finding shapes:

- warn — `docs/CODE_HEALTH_BASELINE.md:101: promote AF-006 —
  exports(src/data/telemetry/queries.ts) = 26 (trigger: >= 26)`
- error — `docs/CODE_HEALTH_BASELINE.md:105: unparseable watch-trigger line:
  'AF-007: loc(...) > 15'` (closed grammar)
- error — trigger subject unmeasurable (missing file / unknown zone), message
  from `MeasureError`.

Multi-line blocks: any line true trips its AF id (grammar semantics).

### 5c. `check_baseline_claims.py` (new in `.agent-local/`)

```python
"""Recomputes the health baseline's mechanically-derivable factual
claims and diffs them against the asserted values. Claims covered:
the five countable Step 1 rows (production files, production LOC,
test files, source suppressions, whole-version clone groups), export
counts on every table row that names a backticked src/convex file with
'N exports' in its text, existence of every src/convex file named in a
baseline table row, and the auth-surface zone file count against the
Rails 'exactly <n> files' phrase. Severities: nonexistent named file =
ERROR (the claim's subject is gone; measurement impossible); stale
number = WARN showing asserted vs measured (counts legitimately drift
mid-session; close-out owns reconciliation); a missing/unparseable
expected Step 1 row = ERROR so a baseline reshape cannot silently
blind the check (the check_env_example precedent). Rows whose
Delta/note marks them 'carried' skip the value diff (existence checks
still run). Coverage, Fallow health score, above-threshold counts, and
judgment rows (hotspot ranking, verdicts, trend) are deliberately out —
they need full runs or judgment and stay audit-owned."""
```

Claim classes:

| # | Claim | Asserted from | Measured by | Severity |
|---|---|---|---|---|
| C1 | 5 countable Step 1 rows | `\| <label> \| <n> \|` (first numeric cell, commas stripped) | `repo_measures` walkers / `clone_file_counts` length | stale = warn; missing/unparseable row = error |
| C2 | Export counts | any table row: backticked `src/…`/`convex/…` + `(\d+) exports` | `export_count` | stale = warn |
| C3 | Named-file existence | every backticked `src/…`/`convex/…` in a baseline table row | `Path.is_file` | missing = error |
| C4 | auth-surface file count | Rails phrase `auth-surface … exactly <digit\|one–ten> files` | `zone_file_count` | stale = warn; phrase absent = warn |

(C3 overlaps `check_doc_refs`' general dead-path sweep by design — this
checker must stand alone at close-out; noted in the docstring.)

## 6. Design alternatives (design-it-twice records)

1. **Measurement ownership** — *chosen:* one shared `repo_measures.py`
   imported by both checkers. *Rejected:* each checker measuring for itself —
   the exports/zone/clone counting rules would live in two files and drift
   (P2), and the fallow subprocess seam would be duplicated.
2. **Clone-count acquisition** — *chosen:* live `fallow dupes --format json`
   subprocess (~0.14 s) with an injectable seam for fixtures. *Rejected:*
   parsing a saved report artifact from the close-out Fallow gate — creates
   artifact-freshness coupling and a new unmanaged intermediate file for zero
   speed benefit.
3. **Trigger-fence parsing** — *chosen:* the expression parser/evaluator is
   new and owned by `check_watch_triggers.py`; `check_lifecycle_evidence.py`
   keeps its id-only scan. *Rejected:* extracting a shared fence parser —
   churns a just-shipped checker to share ~6 boring lines while the two
   owners answer different questions (id symmetry vs. expression evaluation).
4. **Drift-gate membership** — *chosen:* fixtures auto-join the drift gate
   (existing `test_*.py` glob); the live checkers wire into the **close-out
   sequence only**, per the contract. *Rejected:* adding both to
   `check_agent_drift.py`'s live-checker list — claims warns legitimately
   accumulate mid-session and would make every routine drift run noisy for
   findings only close-out can act on.
5. **Boundary mechanism** — *chosen:* targeted ignore list (track-by-default
   inside the workspace trees). *Rejected:* keeping broad ignores plus
   `!`-negation allowlists per tracked file — every new skill/doc would need
   a `.gitignore` edit, and a forgotten negation silently hides work from
   review (the exact failure this change eliminates).
6. **Delivery shape** — *chosen:* one sub-version, one PR (boundary import +
   checkers together); the import is docs/config, and Greptile reviews the
   finished workflow system once. *Rejected:* a separate boundary-only
   sub-version first — two PR cycles and a contract renumber for work whose
   whole purpose is to make *this* sub-version reviewable.

## 7. Contract decisions resolved

- **Tolerance policy:** adopted as proposed — the checker is context-free and
  always reports stale numbers as warns; exactness is enforced procedurally
  at close-out (SESSION_END text: reconcile or explain every warn before a
  final-session PR). No numeric tolerance windows in code.
- **Carried `Targeted:` rows:** simplified from the proposal — a row whose
  note marks it carried always skips the value diff (existence checks still
  run). The "unless its named surface is in the diff" refinement is rejected:
  pre-PR design review already owns remeasuring changed hotspot surfaces in
  the same change, so a git-diff-aware special case here would duplicate that
  owner (P2) for marginal value.
- **Boundary decisions (operator-resolved during re-planning):** first names
  accepted in tracked docs; SCRATCHPAD + backlog tracked; margin-audit and
  capture artifacts stay private.
- **Tracked delivery evidence (operator-resolved during execution):** reconcile
  roadmap/session/SCRATCHPAD truth locally after merge and carry it as the first
  commit on the next lifecycle branch. The remote artifacts intentionally lag by
  one PR; no follow-up PR or direct-to-main push exists solely for reconciliation.
