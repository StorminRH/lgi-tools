Wireframe Notes — landing-grid

Placeholder logic. One tile per anticipated tool, always rendered. Distinguished only by the LIVE / SOON badge + footer copy + interactivity. The first tile (Wormhole Sites) is drawn in its hovered state so the green top-border, tinted background, and soft outer glow are visible without a real pointer. Real hover transitions in 150ms on border-color, background, and box-shadow.

SOON treatment. cursor: default, description at 60% opacity, no hover-glow. The badge alone signals interactivity — no separate dim/disabled state on the title.

Promoting a tool. Swap the badge to LIVE, replace the phase footer with a live freshness chip (prices Xh ago), and make the tile clickable. Removing a backlog item = delete the tile. No layout work.

Tokens demonstrated: --text-muted-2: #6a7a8a (footer copy, freshness, section labels, dim nav-tools) replaces the current #3a5060 for a readability lift. The original token stays in :root so the user can compare side-by-side. Link hover-to-green applies to every <a> outside tiles — see the version link in the footer.
