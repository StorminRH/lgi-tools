Wireframe Notes — nav-search-overlay-takeover

Mechanic. The search starts at the same 280px inline position as inline-push. On focus, instead of pushing tools, the search field becomes absolutely positioned and expands to cover the entire nav from just-right-of-the-wordmark to the right edge — including over the tools and the login chip, which fade out behind it (opacity 0.15, slight blur, pointer-events disabled). The dropdown spans the same width as the expanded search, allowing a 2-column result grid that doubles the visible result count without doubling the dropdown height.

Cursor handoff. Same as inline-push: wordmark cursor drops to 60% opacity and stops blinking; search caret animates. The wordmark itself stays at full opacity — only the cursor handoff happens.

Scope chips live in the search bar. When the search is active, a row of scope chips appears inside the input row (all selected, tools / sites / resources / commands available). Clicking one filters the dropdown immediately. This is the Spotlight pattern.

Pros.

  • Search gets the full visual stage — most "important thing on screen" of any nav option.
  • 2-column result grid uses the extra width to show double the matches without scrolling.
  • Scope chips are visible inline — no need to click a separate filter UI.
  • Fading (not removing) the tools means they're still discoverable: a glance and the user sees what else is available, even with search open.

Cons / open questions.

  • The login chip (admin actions, character ID) is hidden under the search overlay — confirm this is acceptable when admin actions are mid-flight (e.g., role-toggling).
  • The animation is more dramatic (full-width expansion). Tune the easing — too fast and it feels nervous, too slow and it feels heavy.
  • Esc-to-close needs to feel snappy; ensure the "esc" hint chip is visible at the right edge of the expanded search at all times.

The dropdown footer states the search scope explicitly so a first-time user understands what they're searching: "tools · 69 sites · SDE ore + gas · recent · commands". This is the Mac Spotlight footer equivalent ("searching: Applications, Documents, …").
