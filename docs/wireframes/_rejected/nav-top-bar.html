Wireframe Notes — nav-top-bar (Recommended)

Layout. Sticky 44px bar. Left: wordmark (clickable, returns to /) + tool strip showing Wormhole Sites active (green underline + bright text), Industry drawn in hovered state (color shifted to --isk to demonstrate the universal link-hover rule), Blueprint Calc at rest. Right: freshness indicator + login chip (admin link, portrait, name, characterId pill).

Right-slot collision check. Freshness + full login chip (admin chip + portrait + name + characterId) fit at 44px when the name is ≤16 chars. Watch out: very long character names will push the freshness indicator into the tool strip on narrower viewports. If this becomes a real issue, the fallback is to drop freshness from the global nav (it already exists in RefreshFooter on /sites) or expand to 52px height.

Active state vs hover. Active is permanent (green underline + bright --text-name); hover is transient (color-only shift to --isk). They visually coexist — the active tool can't be re-hovered into a different state, which is the point.

Card hover demo. Second ghost site card below the nav is drawn in its hover state: 1px green-tinged border shift + faint outer glow (box-shadow: 0 0 0 1px rgba(61,214,140,0.08)). Lower intensity than the tile hover on the landing page — site cards are dense (sometimes 20+ on screen), so the glow stays restrained.

Same token set as landing-grid.html. The freshness indicator's "prices updated 3h ago" copy is the existing RefreshFooter language lifted into the nav. Once this nav ships, the in-page RefreshFooter can keep its refresh button but drop the timestamp line — the global nav owns it.
