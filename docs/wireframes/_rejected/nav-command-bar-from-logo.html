Wireframe Notes — nav-command-bar-from-logo (Hybrid)

The concept. The terminal cursor █ already in the wordmark IS the entry point. On ⌘K (or click on the cursor block), the command panel drops out of the wordmark like a CLI prompt opening — a green connector bar extends from the cursor downward, the panel's top-left dock-aligns so its border picks up exactly where the cursor lands, and the caret inside the search input is the same green █ as the one in the logo. The user reads it as: "the cursor that's always there opened up."

Layout. Standard 44px top nav (wordmark + tool strip + freshness + login). Tools listed in the nav: Wormhole Sites (active), Industry Planner, Wormhole Roll Calc. The command panel anchors to the wordmark's left edge and is 540px wide; its top-left corner is square (not rounded) so it docks cleanly to the connector. Two-tier glow: green border accent + a wider 0 0 32px -8px rgba(61,214,140,0.25) halo so it reads as the only interactive thing on the page.

Why this beats both standalone options. The plain top-bar nav requires the user to click a tool name to navigate; the plain command-bar requires keyboard discovery. This hybrid does both — the nav tools are still visible and clickable, AND the cursor advertises a keyboard-driven super-power right in the brand mark. New users see a normal nav; power users see the cursor blink and try ⌘K. The terminal-stamp logo finally does something instead of just signalling tone.

Implementation notes. The panel reuses the existing <Modal> primitive's open/close/Esc contract but with anchored positioning instead of centered (so it'll need an anchor prop or a thin subclass — small lift). The search input is the existing <TerminalSearch> primitive promoted from feature-scoped to global. The connector bar is a fixed-position <div> with a green box-shadow; pin coordinates (left: 134px in this wireframe) come from the wordmark SVG's viewBox math — re-derive once if the wordmark's final SVG dimensions change.

Tradeoffs to call out: (1) the connector is brittle if the wordmark is responsive — a fluid nav height or a different wordmark size would shift the anchor. Mitigation: derive the connector's left/width from the actual rendered SVG with one useLayoutEffect. (2) Esc-to-close + click-outside-to-close must be wired the same as <Modal> to keep keyboard discipline. (3) The cursor's blink animation is more prominent now (it's an affordance, not just decoration) — confirm it's not distracting at rest.
