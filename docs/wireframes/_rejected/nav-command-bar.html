Wireframe Notes — nav-command-bar (Alt)

Layout. Persistent chrome shrinks to a 24px header with just the bracket-mark logo and a "⌘K to navigate" hint on the right. Everything else lives behind a modal overlay (this wireframe draws it open by default so the user can see the contents). The overlay reuses the existing <Modal> primitive's contract — centered card, darkening scrim, Esc to close — and the input field is the existing <TerminalSearch> primitive promoted from feature-scoped to global.

Overlay contents. Two result sections — Tools (Wormhole Sites active, Industry + Blueprint Calc as SOON pills) and Recent (last 3 site visits with class-coded icon, name, type, ISK). The active row uses the same green-tinted glow vocabulary as a hovered tile — visual consistency between hover state and keyboard-focus state. Footer carries the existing TerminalSearch syntax hint (c5/relic, c2/combat, …) plus keyboard shortcuts.

Pros: minimal visible chrome (24px gains 20px of vertical content over the top-bar option), and the same affordance handles tool navigation AND in-tool search — one mental model instead of two. Reuses existing primitives; very little net new code.

Cons: discoverability for a first-time visitor is the entire game. The 24px hint has to do all the work of advertising the feature — if the user doesn't try ⌘K, they're stuck. There's no fallback nav. Login chip has nowhere to live except the overlay, which means every status check requires a keystroke.

Recommended only if the user prefers near-invisible nav AND is willing to invest in onboarding copy — a hint chip on first visit, a "Press ⌘K" overlay until dismissed, etc. For a casual EVE Online tool meant to be approachable on first arrival, this is probably too power-user. The top-bar option is a safer fit.
