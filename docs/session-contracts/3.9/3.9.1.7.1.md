## Session 3.9.1.7.1 — Comment standard migration (conversion sweep + worklist)

**Sub-version:** 3.9.1.7 (Phase 1 — Workflow mechanization; session 1 of 2, one branch)
**Master plan:** `docs/VERSION_3_9_PLAN.md` §3.9.1.7
**UX gate:** No

## 1. Objective

First half of the one-style comment migration: every existing `//` comment
block above an exported symbol is converted to `/** */` with its prose
preserved verbatim, and a lint dry-run produces the uncommented-export
worklist that session .2 authors from.

## 2. Current context and dependencies

- Depends on 3.9.1.1 (the standard must be specified before it is enforced).
- The whole 3.9.1.7 slice runs early after 3.9.1.1 — preferably before Phase 2
  begins, so every new primitive is born commented-first under a green lint.
- Session .2 (authoring + enforcement flip) shares this branch; this session
  ends with a verified commit/push, not a PR.

## 3. Done conditions

- Every existing `//` comment block sitting above an exported symbol in `src/`
  and `convex/` production code is converted to `/** */` with its prose
  **preserved verbatim** — the migration changes syntax, never words. Module
  prologues and inner rationale comments stay `//` untouched.
- The conversion is tooling-assisted: a workspace-local script drafts the
  syntax flip and the agent reviews every hunk.
- A lint dry-run (presence rule scoped to exported declarations, TSDoc
  syntax, TODO ban) produces the complete uncommented-export worklist for
  session .2, plus the inventory of stray `TODO`/`FIXME` comments to
  disposition.
- Zero executable-code changes: the diff is comments-only (plus the workspace
  tooling, which is deleted at close per scratch-script hygiene).

## 4. In scope

`src/` and `convex/` production exports (conversion half); the sweep tooling
(workspace-local); the dry-run worklist artifact.

## 5. Out of scope

- Rewording any existing comment (verbatim-preservation is a hard constraint).
- Authoring new comments for uncommented exports (session .2).
- Flipping the lint rules on (session .2).
- Test files and `scripts/` (exempt by default — session plan confirms or
  adjusts the exemption set).
- Any code change.

## 6. Hard constraints

- Verbatim means verbatim: no rewording, and (proposed) no line-wrap
  normalization — the session plan confirms.
- One comment style, no dual-style period *at the sub-version boundary*: the
  branch merges only after session .2 flips enforcement on repo-wide.
- `pnpm verify` green with an unchanged test count.
- Suppression count must not grow.

## 7. Decisions the session plan must resolve

- The exact export-detection scope for the presence rule (declaration exports
  vs. re-exports vs. type-only exports).
- The exemption set (tests, `scripts/`, generated Convex files — the five
  generated-header suppressions must not trip TSDoc validation).
- Whether the conversion script normalizes comment line-wrapping (proposed:
  no).

## 8. Acceptance criteria

- A spot-diff sample proves verbatim conversion (syntax-only changes).
- The uncommented-export worklist exists and is complete per the dry-run.
- Module prologues and inner `//` rationale comments are untouched.
- Unchanged test count; comments-plus-tooling-only diff.

## 9. Verification

- `pnpm verify` green with unchanged test count.
- The dry-run output (rule → violation count → worklist) recorded.
- Verified commit/push on the 3.9.1.7 branch.

## 10. UX/operator gates

**UX gate: No.**

## 11. Baseline/hotspot boundary

Neutral — comments only; suppression count must not grow (a file that can't
satisfy TSDoc syntax gets fixed, not suppressed). No hotspot contact.
No-campaign decision preserved.

## 12. Close-out behavior

Mid-slice session: ends with a verified commit/push to the shared 3.9.1.7
branch and updated session memory — **no PR**. Session .2 completes the
sub-version and opens the single PR.
