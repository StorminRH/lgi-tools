**UX gate:** No

## 1. Objective

The standing claim "Convex state is fully derived — teardown + resync
reproduces it" is verified by doing it, not asserted.

## 2. Current context and dependencies

- No dependencies; pairs naturally with 3.9.3.7's measurement cells in
  scheduling. Never touches production Convex.
- Read `docs/CONVEX.md` before changing anything Convex-adjacent (standing
  rule).

## 3. Done conditions

- Against the dev (or a disposable preview) deployment: tear down the Convex
  state, run the documented resync path, and verify the live surfaces
  reconverge (sync subjects re-arm, projections repopulate, the sweeper
  reports healthy).
- Any manual step the drill uncovers is either automated or written into the
  runbook.
- Any state that turns out *not* to be regenerable is a finding reported to
  Ryan (it would contradict a durable invariant).

## 4. In scope

The drill on non-production state, the runbook, findings.

## 5. Out of scope

- Production Convex.
- Engine behavior changes.

## 6. Hard constraints

- Arc-wide (Phase 4): disposable targets only (dev/preview deployment);
  secrets never enter the runbook; written runbook + timed result in
  SCRATCHPAD; operator-mode steps explicit (local Convex dev requires the
  signed-in operator).
- Neon is authoritative; Convex is derived — the drill verifies, never
  inverts, that relationship.

## 7. Decisions the session plan must resolve

- Which deployment hosts the drill (local dev vs. disposable preview) and the
  teardown method.
- The reconvergence checklist (which surfaces/subjects prove convergence).

## 8. Acceptance criteria

- Before/after state evidence: teardown observed, resync run, reconvergence
  verified (subjects re-armed, projections repopulated, sweeper healthy).
- Every manual step is automated or in the runbook.
- Any non-regenerable state reported as a finding.

## 9. Verification

- The reconvergence readout + committed runbook.
- SCRATCHPAD records the timed result.
- Changelog entry + APP_VERSION bump (3.9.4.4) for tracked changes; otherwise
  the non-code completion shape.
