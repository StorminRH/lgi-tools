## Session 3.9.1.4 — Baseline claims & Watch tripwires

**Sub-version:** 3.9.1.4 (Phase 1 — Workflow mechanization; one session)

**UX gate:** No

## 1. Objective

The baseline's cheap factual claims are recomputed rather than trusted, and
Watch findings fire the moment their trigger trips instead of waiting for the
next audit to notice.

## 2. Current context and dependencies

- Depends on 3.9.1.1 (trigger grammar) and 3.9.1.3 (shared parsing helpers).
- The live Watch set is AF-006–AF-009 in `docs/CODE_HEALTH_BASELINE.md`;
  AF-009's trigger is additionally mooted once 3.9.2.2 lands.

## 3. Done conditions

- `check_baseline_claims.py` recomputes the mechanically-derivable Step 1 rows
  (production/test file counts, LOC, export counts on the known-wide surfaces
  the baseline names, suppression count, `auth-surface` file count,
  clone-group count) and diffs them against the baseline's asserted values. A
  baseline row referencing a nonexistent file is an **error**; a stale number
  is a **warn** with both values shown. Coverage and Fallow score are
  explicitly out — they need full runs and stay audit-owned.
- `check_watch_triggers.py` evaluates every trigger block in the baseline's
  Watch rows and prints `promote AF-NNN` with the measured value when tripped.
  The initial concrete set is AF-006–AF-009 (26th telemetry export; >15
  refresh-jobs exports; a fourth `auth-surface` file; a third cron-shell
  clone).
- Both wire into the close-out sequence (and are runnable standalone); a
  tripped Watch trigger is a **warn** that close-out must surface to Ryan,
  never an auto-promotion — classification stays an audit decision (P10).

## 6. Hard constraints

- Arc-wide (Phase 1): Python 3 stdlib only; `--check`/`--pretty`; wire into
  close-out, never a new standalone ritual; error vs. warn; fixtures per the
  established pattern; no application-code behavior change.

  auto-promote findings.

## 7. Decisions the session plan must resolve

## 8. Acceptance criteria

- A fixture proves a deleted-file claim is caught as an error.
- A fixture proves a tripped trigger prints `promote AF-NNN` with the measured
  value.
- Stale-number findings show both asserted and measured values as warns.
- Close-out procedure text names both checks as steps.
- The agent workspace is tracked; machine-path and secret sweeps over the
  staged set are clean and recorded; `.claude/settings.local.json` and the
  other local-state paths remain git-ignored; the drift gate is green with
  the inverted ignored-set policy; the exposed Neon API key is scrubbed from
  the allowlist and rotated.
