## Session 3.9.2.5 — ux-check probe harness (Combine)

**Sub-version:** 3.9.2.5 (Phase 2 — Primitive lifecycle; one session)
**Master plan:** `docs/VERSION_3_9_PLAN.md` §3.9.2.5
**UX gate:** No

## 1. Objective

The ~30 one-off probe scripts under `docs/ux-check/` collapse into one
probe-definition primitive, making browser probes as repeatable for the agent
as `ux-capture` sweeps already are.

**Verdict & evidence.** *Combine.* ~30 ad-hoc `*-probe.mjs` scripts
re-implement launch/navigate/wait/console-collect/assert around
`ux-capture.mjs`'s core — the literal one-off-script smell (several are
versioned near-duplicates: `hero-two-groups-probe` 1/2/3). Agent-tools area:
this is workspace tooling, not app code.

## 2. Current context and dependencies

- No dependencies; independent of 3.9.2.1–4; may interleave anywhere.

## 3. Done conditions

- One probe runner owns browser lifecycle, console/network capture, and
  reporting; a probe is a small declarative definition (route, setup
  interactions, assertions).
- The recurring probe families (dialog/overlay open, CSP, network,
  element-state) become parameterized definitions; superseded one-offs are
  deleted.
- `docs/ux-check/README.md` documents the definition format and the ux-check
  skill (both runtimes) points at it.
- One-off exploration remains allowed — the harness makes the *durable*
  probes cheap; scratch scripts stay allowed and are deleted at close-out per
  existing workspace hygiene.

## 4. In scope

`docs/ux-check/` scripts + README, the paired ux-check skills (both runtimes,
manifest revision bump), the drift-gate path check, deletion of superseded
probes.

## 5. Out of scope

- `scripts/ux-capture.mjs` behavior (the sweep contract is consumed as-is).
- Adding Playwright to the app's test stack.
- Any app code.

## 6. Hard constraints

- **Rail (workspace-side; lint can't reach gitignored docs):** a drift-gate
  path check flags `*-probe.mjs` files outside the runner's definitions
  layout, and the paired ux-check skills instruct the runner-first flow —
  scratch scripts stay allowed but are flagged for close-out deletion.
- P4 applied to tooling: only probe families with real recurrence get
  definitions; genuine one-shots are deleted unreplaced.
- Skill-pair parity + manifest revision bump in the same change.

## 7. Decisions the session plan must resolve

- Definition format (data file vs. tiny JS module per probe).
- Which probe families have real recurrence vs. genuine one-shots to delete
  unreplaced.

## 8. Acceptance criteria

- Each retained probe family demonstrated via the runner.
- Superseded one-offs deleted; probe count and total probe LOC before/after
  recorded.
- The drift-gate path check flags a seeded stray `*-probe.mjs`.

## 9. Verification

- Probe count/LOC before-after readout in the notes.
- Changelog entry + APP_VERSION bump (3.9.2.5); `pnpm verify` green
  (workspace-only change; still run before commit).
