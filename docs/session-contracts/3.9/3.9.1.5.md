## Session 3.9.1.5 — Close-out gate mechanization

**Sub-version:** 3.9.1.5 (Phase 1 — Workflow mechanization; one session)
**Master plan:** `docs/VERSION_3_9_PLAN.md` §3.9.1.5
**UX gate:** No

## 1. Objective

The three close-out rituals that currently rely on read-back-and-hope —
release consistency, PR-body privacy, and the archive transition — get
mechanical verification.

## 2. Current context and dependencies

- Depends on 3.9.1.1 (vocabulary); independent of 3.9.1.3/.4.
- The Greptile loop and `merge_clean_pr.py` are already mechanized and are not
  changed here.

## 3. Done conditions

- `check_release_consistency.py`: `APP_VERSION`, the newest heading in the
  version's `content/changelog/vX.Y.md`, and the roadmap's latest terminal row
  agree before a PR opens; mismatch is an error.
- `scrub_pr_body.py`: scans a PR body-file for personal names, emails, local
  absolute paths, machine/profile identifiers, and credential-shaped strings
  per the PR_REVIEW privacy rule; findings block publish; the pattern set is
  maintained in the script with a documented extension point.
- `verify_archive.py`: mechanizes the seven archive preconditions from
  `docs/DEVELOPMENT_LIFECYCLE.md` §7 and, after the copy, verifies the
  destination bundle contains the plan, contract set, session plans, and audit
  plan byte-identical to the active copies.
- All three are named steps in the close-out / archive procedures and the
  close-out skill (both runtimes); fixtures included.

## 4. In scope

The three scripts, fixtures, SESSION_END/PR_REVIEW/lifecycle wiring sentences,
manifest revision bump.

## 5. Out of scope

- Changing the Greptile loop or `merge_clean_pr.py` (already mechanized).
- Any auto-publish or auto-archive — scripts verify, humans/skills act.

## 6. Hard constraints

- Arc-wide (Phase 1): Python 3 stdlib only; `--check`/`--pretty`; wire into
  close-out/archive procedures, never a new standalone ritual; error vs. warn;
  fixtures per the established pattern; no application-code behavior change.
- Standing decision: checkers report/verify; humans and skills act.

## 7. Decisions the session plan must resolve

- Where the personal-identifier pattern list lives so it isn't
  secret-adjacent content in a tracked file (proposed: in the gitignored
  script itself).
- Whether `verify_archive.py` takes the archive root as a required argument or
  reads a configured path.

## 8. Acceptance criteria

- Each script demonstrated red against a seeded-violation fixture
  (version/changelog/roadmap mismatch; a PR body with a seeded personal
  identifier; an archive bundle missing or differing from an active copy).
- Release-consistency check runs green on this slice's own PR.
- Close-out and archive procedure texts name the three scripts as steps.

## 9. Verification

- `python3 .agent-local/check_agent_drift.py` green at the bumped revision.
- Changelog entry + APP_VERSION bump (3.9.1.5); `pnpm verify` green.
