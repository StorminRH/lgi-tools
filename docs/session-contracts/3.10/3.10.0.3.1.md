## Session 3.10.0.3.1 — Canonical close-out and single-pass verification

**Sub-version:** 3.10.0.3 (Phase 0 — Documentation & lifecycle consolidation; session A of two, one branch)
**Master plan:** `docs/VERSION_3_10_PLAN.md` §3.10.0.3
**UX gate:** No

## 1. Objective

Author the whole close-out workflow as one canonical procedure and remove the
duplicated local Vitest/Fallow cycle by making `pnpm verify` the sole
coverage-backed definition-of-done command on the finalized head.

## 2. Current context and dependencies

- **DEP-1:** Depends on 3.10.0.2 (the `docs/workflows/` frame and canonical-form pattern
  exist). Session A of sub-version 3.10.0.3; session B (`3.10.0.3.2`) cuts over
  the skills and deletes the sources on the same branch.
- **DEP-2:** Today the close-out flow is spread across `SESSION_END`, `SELF_REVIEW`, and
  `PR_REVIEW`, with a hand-rolled Greptile poll and merge recipe duplicating
  `poll_pr_gate.py` / `merge_clean_pr.py`.
- **DEP-3:** `pnpm verify` currently runs non-coverage Vitest and ordinary close-out
  runs a second coverage-enabled Vitest suite before pinned Fallow; CI already
  demonstrates the desired one-coverage-suite order without needing a command change.
- **DEP-4:** The operator-authored `.agent-local/codegraph_guard.py` improvement is
  inherited work and must be delivered with syntax, behavior, parity, and diff proof
  without silently rewriting its wording.

## 3. Done conditions

- **DC-1:** `docs/workflows/close-out.md` exists and merges `SESSION_END` + `SELF_REVIEW` +
  `PR_REVIEW` into one canonical procedure written as actionable steps in the
  real sequence: end-of-session gates, the judgment review (SELF_REVIEW's checks
  as mandatory numbered steps, not self-declared companion material), and the PR
  loop.
- **DC-2:** The hand-rolled Greptile poll and merge recipes are replaced by citing
  `poll_pr_gate.py` / `merge_clean_pr.py`; unchanged-head evidence is reused at PR
  open and later changes rerun only the gates they invalidate; the
  3.0/3.7-era narratives shrink to one line plus an archive pointer.
- **DC-3:** Recorded decisions are baked in: the contract `UX gate` marker is the pause
  authority with judgment as the off-lifecycle fallback (A2); `merge_clean_pr.py`
  is named the gate of record and the Greptile/CI/mergeability checklist stops
  being restated in prose (A3); final sessions mark `Execution status: Complete`
  only after merge (A10).
- **DC-4:** `package.json` defines `verify` exactly as typecheck, zero-warning lint,
  one coverage-enabled Vitest run, and Fallow; the canonical procedure owns the
  one `origin/main`-pinned invocation and no live ordinary-close-out owner retains
  the old split sequence.
- **DC-5:** The operator-authored Codegraph hook edit is complete and passes syntax,
  bash-search/source-read smoke, runtime parity, and drift checks.
