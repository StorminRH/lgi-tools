## Session 3.10.2.1.3 — Cross-runtime seams, enforcement gates, and old-path deletion

**Master plan:** `docs/VERSION_3_10_PLAN.md` §3.10.2.1
**UX gate:** No
**Execution profile:** Frontier autonomous coding agent
**Delivery unit:** One agent session, one shared sub-version branch, one PR per session

## 1. Objective

The end-to-end type-safety chain closes. Convex-to-Next internal calls
consume the same endpoint contracts with runtime validation, timeout, and
service auth preserved; Better Auth's hand-pinned seams are replaced by
official typed client APIs or mechanically bound adapters; query and dynamic
path parameters come from schema-derived builders; the completeness and lint
gates reject every bypass with seeded proof; and the old contract path is
deleted so exactly one system remains. This is the sub-version's final
session; its own PR publishes the planned version records.

## 2. Current context and dependencies

- **DEP-1:** The two prior sessions shipped the typed primitives and migrated every first-party route, endpoint contract, and browser caller onto them; only the seams named here remain.
- **DEP-2:** The Convex seam at planning time is two `fetchWithTimeout` calls in `convex/lib/characterSync.ts` asserting response types from the shared auth contract module, plus the environment-owned base URL; the `convex` zone may import `platform/esi`, `platform/auth`, `data`, and `lib`, but not `transport`.
- **DEP-3:** The Better Auth seam is three hand-pinned endpoint declarations in the auth slice's contract module for sign-out, OAuth sign-in, and token minting, consumed where the auth feature client is not importable; the installed library version owns their real shapes.
- **DEP-4:** The existing completeness gate proves contract association, input markers, and response pins for the v1 world and must be rebuilt around the v2 invariants it now under-enforces.
- **DEP-5:** The dynamic first-party route surface is the wormhole-site detail route plus the query-marker routes; the Better Auth catch-all stays library-owned.

## 3. Done conditions

- **DC-1:** Convex-to-Next internal calls consume the shared endpoint contracts through a typed service transport that applies the trusted base URL, timeout, and service-auth headers, validates responses at runtime, and contains no response type assertion.
- **DC-2:** Each Better Auth hand-pinned seam is replaced by an official typed client API, or bound to exported library types or an executable contract test against the installed version; no unverified assertion or copied interface remains.
- **DC-3:** Query parameters and dynamic path parameters on first-party endpoints are schema-derived and serialized through typed builders; no caller hand-maintains parameter names or encoding.
- **DC-4:** The v1 endpoint model, its `apiFetch` overloads, the legacy wrapper seams, and obsolete allowlists are deleted; exactly one contract system remains.
- **DC-5:** The completeness and lint gates mechanically reject contractless routes, orphaned contracts, raw same-origin fetches, raw internal absolute-URL construction, and response type assertions, each with a seeded red fixture.
- **DC-6:** A full re-run of the seam inventory shows every item disposed: migrated, typed library delegation, external untrusted boundary, or not a programmatic API.
- **DC-7:** This session's PR opens carrying the seam cleanup and the planned version records with behavior preserved, and its as-built record carries the PR number.

## 4. In scope

- **IS-1:** The typed service transport for server-to-server first-party calls and the Convex call-site migration.
- **IS-2:** The Better Auth seam replacement or mechanical binding, including any command-composition restructuring needed to respect the import graph.

## 5. Out of scope

- **OOS-3:** The same-origin enforcement flip and every later roadmap section's scope.
- **OOS-4:** Any new validation library, HTTP client, or router.

## 6. Hard constraints

- **HC-1:** Behavior preservation: statuses, auth ordering, timeouts, service-secret handling, and cron semantics are unchanged by the seam migration and deletion.
- **HC-2:** Repository import-direction rules hold; a service transport home that would require a new cross-zone edge needs explicit operator approval before implementation.
- **HC-3:** No `any`, `as unknown as`, broad assertions, non-null shortcuts, or retained compatibility overloads after final migration.
- **HC-4:** Gate additions must be deterministic and understandable, with every remaining allowlist small, named, justified, and tested against growth.
- **HC-5:** The old path is deleted in this session; this session's PR does not open while both contract systems remain valid.
