## 1. Objective

Reshape the source tree to the structure it would have had if the boundaries
had existed from day one, then make that structure mechanically enforceable
with zero carried-forward exceptions. Cross-slice composition moves above the
slices it composes into a composition band (registry aggregators, the purge
orchestrator, the database sync workers, account-lifecycle orchestration, and
app-shell UI including auth's account components); structural capabilities
move below the slices into `src/platform` (identity and session, the ESI
gate, and the registry socket contracts features implement); shared request
plumbing moves to `src/transport`; the whole online-status slice dissolves
into the data layer, deleting the recorded Convex exception; the tsx entry
scripts get an entry-point home so the database layer stops importing upward.
On the corrected shape, one-way dependency rules hold with no recorded
cross-layer exception of any kind, dependency-cycle detection flips from
advisory to blocking, and a mechanically enforced browser/server rail runs in
verify and lint. This belongs in sub-version 3.10.1.2 because it is the
boundary-hardening step of Phase 1, expanded by explicit operator direction
(2026-07-23): correctness of the structure outranks refactor size, PR size,
and contract continuity. No application behavior changes.
