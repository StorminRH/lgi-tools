## Session 3.10.0.7 — Enforcement follow-through & anti-duplication check

**Sub-version:** 3.10.0.7 (Phase 0 — Documentation & lifecycle consolidation; one session)
**Master plan:** `docs/VERSION_3_10_PLAN.md` §3.10.0.7
**UX gate:** No

## 1. Objective

Make the drift/parity machinery match the consolidated set, and add a check that
makes single-owner mechanical so the sprawl cannot silently regrow. **Phase 0
exit:** an agent's close-out reading path is one dispatched skill → one canonical
procedure in `docs/workflows/`, and no rule has two owners.

## 2. Current context and dependencies

- **DEP-1:** Depends on 3.10.0.3–0.6: the final consolidated set (procedures, thin adapters,
  map docs) must exist to police it.
- **DEP-2:** Each earlier Phase 0 slice already reconciles `policy-manifest.json`
  incrementally in its own change (the bootstrapping discipline); this slice
  performs the **final whole-set audit/rewrite** of `canonicalGuides`/
  `pairedSkills` against the settled corpus and adds the anti-duplication check.

## 3. Done conditions

- **DC-1:** `policy-manifest.json` `canonicalGuides` and `pairedSkills` are rewritten
  against the `docs/workflows/` procedures, the thin adapters, and the map set;
  `skillReconciliation` declares the settled dependency set and is regenerated
  with `reconcile_skill_ledger.py` after the final whole-set review; the
  `check_doc_refs.py` historical/future allowlist is pruned to what the
  consolidated tree still needs.
- **DC-2:** A new drift-gate check fails when the same normative sentence (a hash over
  normalized text) appears in more than one canonical file — spanning the
  `workflows/` procedures, the maps, and the two skill adapters.
- **DC-3:** All `.agent-local/test_*.py` fixtures are updated to match.

## 4. In scope

- **IS-1:** The manifest rewrite, the skill-ledger reconciliation, the allowlist prune, the
anti-duplication check plus its tests, and the fixture updates.

## 5. Out of scope

- **OOS-1:** Any further content change (the corpus is settled by 0.6).
- **OOS-2:** Architecture-phase work (Phases 1–3).

## 6. Hard constraints

- **HC-1:** Exactly one new anti-duplication drift check — no other new parallel
  doc/checker machinery (Phase 0 non-goal).
- **HC-2:** The uniqueness check must not trip on shared rule *names* or adapter
  boilerplate (false-positive tolerance is a resolved decision, §7).
- **HC-3:** No application code changes; green resolver/drift/doc-ref at merge.

## 7. Decisions the session plan must resolve

- **PD-1:** The normalization rule for the duplicate-sentence hash (whitespace/case/
  punctuation folding) and its false-positive tolerance.
- **PD-2:** Which files are in-scope for the uniqueness check vs deliberately shared
  boilerplate.

## 8. Acceptance criteria

- **AC-1:** Proves DC-2: a seeded duplicate normative sentence across two canonical files fails
  `check_agent_drift.py`.
- **AC-2:** Proves DC-3: the full checker suite is green on the consolidated tree.
- **AC-3:** Proves DC-1: `policy-manifest.json` references only the consolidated set; the allowlist is
  pruned.
- **AC-4:** Proves DC-1 and DC-2: **Phase 0 exit condition holds:** the close-out reading path is one skill → one
  `docs/workflows/` procedure, and no rule has two owners.
