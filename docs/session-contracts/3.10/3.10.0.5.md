## 1. Objective

Make the remaining substantive workflow (version-audit) canonical, and make the
skills 0.5 owns — the audit trio plus the currently self-contained
`update-watch`/`ux-check`/`triage-issue` — thin adapters over one procedure. (The
whole-set "every skill is thin" state is Phase 0's exit condition, checked in 0.7.)

## 2. Current context and dependencies

- Depends on 3.10.0.2 (the strict baseline template it emits). Independent of
  0.3/0.4 procedure content.
- `update-watch` is byte-identical across both skill trees today (pure waste);
  `ux-check` and `triage-issue` are also inlined in both trees.

## 3. Done conditions

- `VERSION_AUDIT.md` becomes `docs/workflows/version-audit.md` (its steps; the
  baseline template it emits was extracted in 0.2). The procedure writes the
  baseline in the **strict two-column metrics schema** from 0.2 — it updates the
  current column and records findings/rationale in the version-tagged audit
  report, never as prose in the baseline; at version close it captures the *next*
  version's frozen version-start snapshot (the one sanctioned write of that
  column).
- The `version-audit`, `plan-version-audit`, and `plan-audit-remediation` skills
  are thinned to adapters.
- The three currently-inlined skills are de-duplicated: `update-watch`,
  `ux-check`, and `triage-issue` have their shared procedure extracted to
  `docs/workflows/` with only genuine per-runtime mechanics (capture tooling,
  native syntax) left in each adapter.
- `VERSION_AUDIT.md` is deleted; manifest/refs updated; revision bumped.

## 4. In scope

The version-audit procedure, the audit-skill thinning, the self-contained-trio
de-duplication, the `VERSION_AUDIT.md` retirement, and the manifest updates.

## 5. Out of scope
